## What are Tools?

Tools allow your agents to use your tech stack much like a human would. One of our key philosophies is that subject matter or domain experts need to be the ones to distill their skills into agents. The way we do this is through our low-code tools.

## Tool-building concepts

There are three key concepts to understand when creating tools:

- **Knowledge**: Tools can leverage external knowledge and domain expertise that you have added to the platform via knowledge (or created a tool that does it that for you).

<Frame caption="Example of giving tool access to existing knowledge">
 <img src="/images/tool-knowledge.png" />
</Frame>

- **Inputs**: Tools can accept external input. Make sure the name and description is describes the data and purpose of the data well, so that the agent can figure out what information it needs to pass on to a tool.

<Frame caption="Example of describing inputs a tool receives">
 <img src="/images/tool-inputs.png" />
</Frame>

- **Tool Steps**: You can chain together many different tool steps to get a specific task done.

<Frame caption="Example of describing inputs a tool receives">
 <img src="/images/tool-steps.png" />
</Frame>

## How to create a Tool

To create a new Tool, navigate to the [Tools](/get-started/key-concepts/tools) page and click on the `+ Create tool` button in the top right corner.

You have two options when creating a Tool:

1. Use a [ready-to-use template](/templates/introduction) designed and maintained by Relevance.
2. [Create your own custom Tool](/build-custom-tools/create-a-tool) from scratch.
