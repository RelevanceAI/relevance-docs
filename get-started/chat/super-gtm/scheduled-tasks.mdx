Super GTM can run [skills](/get-started/chat/super-gtm/skills) on a recurring schedule. This is useful for anything that should happen regularly without you needing to ask — daily pipeline reports, weekly account summaries, automated outreach sequences, or periodic data hygiene tasks.

## How scheduling works

When you schedule a skill, Super GTM runs it automatically at the interval you specify. The agent executes the skill exactly as it would in a conversation — pulling from connected integrations, referencing files, applying memory — and delivers the output.

<Info>
Scheduled tasks run in the background. You'll receive the results when the task completes, typically via a notification or a message in your Super GTM inbox.
</Info>

## Setting up a schedule

Schedules are configured in the **Schedules** section of the Super GTM Overview tab.

<Steps>
  <Step title="Open the Schedules section">
    In the Super GTM Overview tab, navigate to the **Schedules** section.
  </Step>
  <Step title="Select a skill">
    Choose the skill you want to run on a schedule. Any skill in your skill library can be scheduled.
  </Step>
  <Step title="Set the frequency">
    Choose how often the skill should run: daily, weekly, or a custom cron-style interval. See the frequency options table below for details.
  </Step>
  <Step title="Configure delivery">
    Specify where the output should be sent — for example, posted to a Slack channel, emailed to you, or saved to a file in the file system.
  </Step>
  <Step title="Activate the schedule">
    Save and enable the schedule. The skill will run automatically at the next scheduled time.
  </Step>
</Steps>

## Recurring schedule options

| Frequency | Description | Example use case |
| --- | --- | --- |
| Daily | Runs every day at a set time | Morning pipeline digest, daily standup prep |
| Weekly | Runs on specific days each week | Weekly account health summary, Monday kick-off report |
| Custom | Cron-style interval | Every Monday and Thursday at 9am, first of each month |

## Managing schedules

<CardGroup cols={3}>
  <Card title="Pause" icon="pause">
    Stop a schedule temporarily without deleting it. Resume at any time.
  </Card>
  <Card title="Edit" icon="pen-to-square">
    Change the skill, frequency, or delivery settings for an existing schedule.
  </Card>
  <Card title="Delete" icon="trash">
    Remove a schedule permanently when it's no longer needed.
  </Card>
</CardGroup>

All schedules are managed from the Schedules section of the Overview tab.

## What scheduled tasks can do

Scheduled tasks run the full skill workflow with access to the same capabilities as a live conversation:

<CardGroup cols={2}>
  <Card title="File system" icon="folder-open">
    Read from and write to the [file system](/get-started/chat/super-gtm/file-system)
  </Card>
  <Card title="Integrations" icon="plug">
    Pull data from connected [integrations](/get-started/chat/super-gtm/integrations) (CRM, calendar, email, etc.)
  </Card>
  <Card title="Memory" icon="brain">
    Apply [memory](/get-started/chat/super-gtm/memory) to personalize output
  </Card>
  <Card title="Sub-agents" icon="robot">
    Run sub-agents from your Relevance AI agent library
  </Card>
</CardGroup>

<Warning>
Scheduled tasks that take write actions — such as sending an email or creating a CRM record — still require approval unless the skill has been explicitly configured for autonomous execution. Check your skill's configuration before scheduling write-heavy workflows.
</Warning>
