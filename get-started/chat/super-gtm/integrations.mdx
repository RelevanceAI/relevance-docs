Super GTM connects to your go-to-market tech stack so the agent can read data and take actions across your tools. Integrations are connected per user (OAuth) or per project (API key), and are managed from the Super GTM settings panel.

## Connecting integrations

To connect an integration:

1. Click the gear icon next to the Super GTM toggle to open the settings panel
2. Navigate to the **Integrations** section
3. Find the integration you want to connect
4. Click **Connect account** (OAuth) or **Add API key** (API key integrations)

### OAuth vs API key

There are two connection methods, depending on the integration:

- **OAuth:** Each user authorizes their own connection via a popup modal. Every team member must connect their own account individually. This is the standard method for tools like HubSpot, Google, and Slack.
- **API key:** A single API key is added to the Relevance project and shared across all users in that project. Admins can manage these centrally in the builders app under [Integrations & API Keys](/get-started/core-concepts/api-integration).

For per-user OAuth setup at the organizational level, see [user-level authentication](/enterprise/user-level-authentication).

<Note>
Depending on how your organization has configured each external application, you may need to request admin access to enable OAuth or to obtain an API key. Check with your IT team or the admin of the external application.
</Note>

## Supported integrations

### CRM & sales

<AccordionGroup>
  <Accordion title="HubSpot">
    **Connection method:** OAuth

    **What it does:** Gives Super GTM read and write access to your HubSpot CRM. The agent can search contacts, companies, and deals; create and update records; and pull activity history to give you a complete picture of any account.

    **Permissions:** Read, create, update, delete, and search CRM objects (contacts, companies, deals, tickets, etc.). List owners.

    **How to connect:**
    1. In the Super GTM settings panel, find **HubSpot** under Integrations
    2. Click **Connect account**
    3. Sign in to HubSpot and authorize access when prompted
    4. Select the HubSpot portal to connect if you have multiple

    **Things to know:**
    - Each user must authorize their own HubSpot connection
    - If your HubSpot portal restricts API access, you may need your HubSpot admin to approve the integration
    - The agent respects your HubSpot user permissions — it can only access records you have permission to see
  </Accordion>

  <Accordion title="Salesforce">
    **Connection method:** OAuth

    **What it does:** Connects Super GTM to your Salesforce org. The agent can read and write CRM data, run SOQL queries to find specific records, and take actions across Salesforce objects.

    **Permissions:** Read and write CRM data via API. Run SOQL queries.

    **How to connect:**
    1. In the Super GTM settings panel, find **Salesforce** under Integrations
    2. Click **Connect account**
    3. Sign in to Salesforce and authorize the connection
    4. If your org requires admin approval for connected apps, request access from your Salesforce admin

    **Things to know:**
    - Salesforce organizations often require admin approval for new OAuth apps — check with your Salesforce admin if the connection fails
    - The agent operates within your Salesforce user permissions and sharing rules
    - Custom objects and fields are accessible if they're visible to your user profile
  </Accordion>

  <Accordion title="Apollo">
    **Connection method:** API key

    **What it does:** Gives Super GTM access to Apollo's sales intelligence database. The agent can search and enrich contact and company data, find contact information, and pull prospect details directly into your workflow.

    **Permissions:** Search, create, and update contacts and accounts. List, create, and update deals. Enrich person and organization data.

    **How to connect:**
    1. Log in to Apollo and navigate to Settings > Integrations > API
    2. Generate an API key
    3. In the Super GTM settings panel, find **Apollo** under Integrations
    4. Click **Add API key** and paste your Apollo API key

    **Things to know:**
    - Apollo API keys are scoped to your Apollo account — each user needs their own key, or an admin can add a shared project-level key
    - API usage counts against your Apollo plan's API limits
  </Accordion>

  <Accordion title="Gong">
    **Connection method:** API key

    **What it does:** Connects Super GTM to Gong call recordings and transcripts. The agent can pull call data, surface key moments, and incorporate conversation intelligence into deal briefings and account summaries.

    **Permissions:** Read call data and recordings via Gong API.

    **How to connect:**
    1. In Gong, navigate to Settings > Ecosystem > API
    2. Generate an API key (requires Gong admin access)
    3. In the Super GTM settings panel, find **Gong** under Integrations
    4. Click **Add API key** and enter your Gong API credentials

    **Things to know:**
    - Gong API access typically requires Technical Admin permissions in Gong — check with your Gong admin
    - The agent can only access calls that are already processed and indexed in Gong
  </Accordion>
</AccordionGroup>

### Communication & collaboration

<AccordionGroup>
  <Accordion title="Slack">

    **Permissions:** Read and post messages. Search messages. Manage channels. List users. Add reactions.

    1. In the Super GTM settings panel, find **Slack** under Integrations

    **Things to know:**
    - Gmail access is part of the GSuite integration, which also covers Google Calendar and Google Docs

    - Sent emails appear in your Gmail Sent folder as normal

## Managing connections

All active integrations are visible in the Super GTM settings panel. You can:

- **Reconnect** an integration if your credentials expire or permissions change
- **Disconnect** an integration to revoke access
- **View permissions** to see what the agent can do with each connected tool

For project-level API key integrations, admins can manage these centrally in the builders app under [Integrations & API Keys](/get-started/core-concepts/api-integration). This is the recommended approach for API key integrations that should be shared across your team.

## Troubleshooting

**Authorization fails or redirects to an error page.** This usually means your organization has restricted third-party app access. Check with your IT admin or the admin of the external application to request approval.

**The agent can't see data I expect it to find.** Confirm that the integration is connected and that your account has access to the relevant data in the external tool. For Notion, check that you've shared the specific pages during the OAuth flow.

**An OAuth connection expires.** OAuth tokens can expire or be revoked. If the agent reports that a tool is disconnected, go to the Integrations section in Super GTM settings and reconnect.

**API key stops working.** API keys can be revoked or rotated. Generate a new key in the external tool and update it in the Super GTM settings panel.
