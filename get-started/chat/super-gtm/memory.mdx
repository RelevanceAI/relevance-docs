---
title: "Memory"
description: "Per-user learning that makes the agent consistent and personal across every conversation"
---

Memory is always on. At the start of every Super GTM conversation, the agent loads everything it knows about you — your preferences, feedback you've given, context about your accounts and role, and resources you've pointed it to. You don't need to re-explain yourself. The agent already knows.

Over time, memory makes the agent progressively more useful. It picks up on how you like things formatted, which accounts matter to you, what's happened in past conversations, and how to work with you effectively.

## How memory loads

When you start a Super GTM session, the agent automatically loads your memory before responding to anything. This happens in the background — you won't see it, but the effect is that the agent already has context when the conversation begins.

<Note>
Memory is scoped per user. Your memory is not shared with other users in your project.
</Note>

## Memory types

Super GTM organizes memory into four categories:

<CardGroup cols={2}>
  <Card title="User memory" icon="user">
    Information about who you are: your role, goals, responsibilities, domain expertise, and how you prefer to work
  </Card>
  <Card title="Feedback memory" icon="comment">
    Guidance you've given about how the agent should approach tasks, including corrections and confirmations
  </Card>
  <Card title="Project memory" icon="briefcase">
    Information about ongoing work: accounts, deals in progress, company context, and active initiatives
  </Card>
  <Card title="Reference memory" icon="bookmark">
    Pointers to where information lives — in files, external systems, or specific locations the agent should check
  </Card>
</CardGroup>

<AccordionGroup>
  <Accordion title="User memory examples">
    - "User is a senior AE focused on mid-market SaaS accounts"
    - "User prefers short, direct email drafts — no filler phrases"
    - "User is new to using CRM and prefers step-by-step guidance"

    The agent uses this to tailor responses — for example, it knows whether to explain a concept from first principles or assume you already know it.
  </Accordion>
  <Accordion title="Feedback memory examples">
    - "Don't include pleasantries in email drafts"
    - "Always include a next-step CTA in outreach"
    - "User confirmed that bullet-point deal summaries are the preferred format"

    Feedback memory is how the agent learns your working style over time. If you correct the agent, it won't make the same mistake next time.
  </Accordion>
  <Accordion title="Project memory examples">
    - "ACME Corp is in late-stage negotiation; decision maker is Sarah Chen"
    - "User's team is launching a new product line in Q3 — mention in relevant outreach"
    - "Competitor X recently raised a Series C — flag in competitive briefings"
  </Accordion>
  <Accordion title="Reference memory examples">
    - "The pricing sheet for enterprise accounts is in the file system under /pricing/enterprise.md"
    - "Pipeline deals are tracked in HubSpot under the 'Q2 Pipeline' view"
  </Accordion>
</AccordionGroup>

## Managing memory

<Tabs>
  <Tab title="Remembering">
    You can explicitly ask the agent to save something to memory at any time:

    1. "Remember that I prefer email drafts in first person"
    2. "Note that ACME Corp's budget cycle runs January to March"
    3. "Remember that we don't use 'synergy' in any customer-facing content"

    The agent will confirm what it stored and which memory type it used.
  </Tab>
  <Tab title="Forgetting">
    If a memory is outdated or incorrect, you can ask the agent to remove it:

    1. "Forget that ACME Corp is in negotiation — that deal closed"
    2. "Remove the note about my preference for bullet-point summaries"
    3. "Clear any memory about Project X"

    The agent will confirm what was removed.
  </Tab>
  <Tab title="Reviewing">
    Periodically review what the agent remembers by asking:

    1. "What do you know about me?"
    2. "What have you saved about ACME Corp?"
    3. "Show me my feedback preferences"

    The agent stores memories to be helpful, not as a permanent record. You can always update or remove entries.
  </Tab>
</Tabs>

## How skills and memory work together

When a [skill](/get-started/chat/super-gtm/skills) runs, it has access to your full memory. This means the same skill produces personalized output for you without needing a separate skill definition for each user.

| Scenario | How memory helps |
| --- | --- |
| Draft outreach email | Writes in your preferred tone if your tone preferences are in memory |
| Deal summary | Highlights the details you care about if your priorities are stored |
| Account research | Incorporates known context about the account from project memory |

Skills can also be written to update memory as part of their workflow — for example, "note any new information learned about this account." Over time, this makes your project memory richer and more accurate.

## Keeping memory accurate

<Tip>
Memory is only useful if it's current. Follow these practices to keep it reliable.
</Tip>

<Steps>
  <Step title="Correct mistakes immediately">
    When the agent misremembers or applies outdated context, correct it. This updates feedback memory.
  </Step>
  <Step title="Update when situations change">
    When a deal closes, a contact changes roles, or priorities shift — tell the agent so it can update project memory.
  </Step>
  <Step title="Review periodically">
    Ask the agent what it knows about you or a specific account to catch stale information before it affects output.
  </Step>
</Steps>
