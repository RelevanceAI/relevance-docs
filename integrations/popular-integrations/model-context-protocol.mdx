The MCP (Model Context Protocol) integration lets you connect tools to and from Relevance AI. Give your agents access to tools hosted on external MCP servers through preset connections (Notion, Canva, Atlassian) or custom server URLs, or expose your Relevance AI tools to services like Claude and Cursor. This means you can centralize your tool management in one place while extending their reach across your workflow — or pull in capabilities from other platforms without rebuilding them.

There are two ways to use MCP with Relevance AI:

1. **External MCP server → Relevance AI Agent**: Connect a remote MCP server to your agent, giving it access to tools from another service.
2. **Relevance AI tools → External service**: Expose your Relevance AI tools to an external MCP-compatible client like Claude or Cursor.

## Setting Up MCP Server Integration

<Note>MCP uses a unified interface across both agent-level and project-level configuration, providing a consistent experience wherever you connect MCP servers.</Note>

### For Agents

Agents can connect to and run tools from remote MCP servers. You can choose between preset connections for popular services or connect your own custom MCP server.

1. Open the Agent you want to connect
2. Navigate to the "Tools" tab in the agent builder
3. Click "Add MCP"
4. Choose your connection type:
   - **Preset connections**: Select from Notion, Canva, or Atlassian for simplified setup
   - **Custom connection**: Click "Connect my own" to add any other MCP server
5. For preset connections:
   - Select the service you want to connect
   - Authenticate with the respective service when prompted
   - The server URL and configuration are handled automatically
6. For custom connections:
   - Add the URL of the remote hosted MCP server
   - Label your MCP Server connection
   - Add any authentication details required for the MCP server

Once connected, the agent will automatically fetch and display the available tools from the remote MCP server. These tools can be referenced in the agent's core instructions and will appear in the conversation task view. When a tool is run, the agent communicates with the remote MCP server and returns the result directly in the conversation.

<Warning>Local MCP server support (e.g. JSON configuration) is not available.</Warning>

Relevance AI provides preset MCP connections for popular services, simplifying the setup process and ensuring secure, reliable integration.

#### Benefits of Preset Connections

- **Simplified setup**: No need to manually configure server URLs or connection parameters
- **Automatic configuration**: Server endpoints and authentication flows are handled for you
- **Secure authentication**: OAuth flows and secure token management built-in
- **Maintained by Relevance AI**: Presets are kept up-to-date with the latest API changes
- **Quick start**: Connect in seconds with just authentication

Custom connections remain available for any other MCP servers not covered by presets, giving you flexibility to connect to any MCP-compatible service.

### For Tools

You can access and use your Relevance AI tools via an MCP Server connection.

1. Go to your Relevance AI workspace
2. Click "More" on the left-side navigation menu, navigate to "MCP servers"
3. Select the tools you want to share with external clients
4. Click "Generate MCP Server Configuration"
5. Copy your MCP configuration into your preferred client

<Tip>
  For Cursor: Open Cursor settings → MCP tab → "Add new MCP server" → Paste into your mcp.json file
</Tip>

## Client-Specific Setup

### Cursor

To use your Relevance AI tools in Cursor:

1. Open Cursor settings
2. Navigate to the MCP tab
3. Click "Add new MCP server"
4. Paste your generated MCP configuration into your mcp.json file
5. Save the configuration

### Claude

To use your Relevance AI tools in Claude:

1. Access Claude's settings
2. Look for the tool or API integration section
3. Add a new MCP server
4. Paste your generated MCP configuration
5. Save the settings

## Best Practices

- **Use preset connections when available**: Preset connections for Notion, Canva, and Atlassian provide simplified setup and automatic maintenance
- **Select tools strategically**: Only expose tools that would be useful in external contexts
- **Test the integration**: After setting up, test each tool to ensure it works as expected
- **Update regularly**: When you update tools in Relevance AI, regenerate your MCP configuration
- **Monitor usage**: Keep track of how your tools are being used across different platforms
- **Secure custom connections**: For custom MCP servers, ensure authentication credentials are properly secured

## Security Considerations

The MCP Server integration maintains the security of your tools while making them accessible to external clients. Authentication is handled through secure tokens, ensuring that only authorized users can access your tools. All communication between Relevance AI and external clients is encrypted to protect your data.

For preset connections, OAuth flows and token management are handled automatically, providing an additional layer of security without requiring manual credential management.

## Troubleshooting

If you encounter issues with the MCP Server integration:

- Verify that your MCP configuration is correctly copied into the external client
- Ensure the external client supports the MCP standard
- Check that the selected tools are compatible with the external client
- Confirm your authentication credentials are valid
- For preset connections, try disconnecting and reconnecting to refresh authentication
- Regenerate the MCP configuration if necessary
