The Relevance AI MCP server gives any MCP-compatible AI client direct access to your agents, tools, and knowledge. Connect from the AI tools you already use and start building.

The MCP server is available at:

```
https://mcp.relevanceai.com/
```

<Tip>For Claude Code, we recommend using the [Relevance AI plugin](/integrations/mcp/programmatic-gtm/claude-code) instead of a manual MCP connection — it includes built-in skills and context that make the experience significantly better.</Tip>

<Warning>
  This page is about using Relevance AI **from** external AI clients. If you want to connect an external MCP server **to** a Relevance AI agent, see [MCP Client](/integrations/mcp/mcp-client).
</Warning>

## Authentication

When you first connect, you will be prompted to authenticate with your Relevance AI account. Authentication is **per project** — you will be connected to a specific Relevance AI project after logging in.

### Working with multiple projects

If you work across multiple Relevance AI projects, add a separate MCP server entry for each:

<CodeGroup>
```json Cursor
{
  "mcpServers": {
    "relevance-project-1": {
      "command": "npx",
      "args": ["-y", "mcp-remote", "https://mcp.relevanceai.com/"]
    },
    "relevance-project-2": {
      "command": "npx",
      "args": ["-y", "mcp-remote", "https://mcp.relevanceai.com/"]
    }
  }
}
```

```json VS Code

    "relevance-project-2": {
```

```json Windsurf
{
  "mcpServers": {
    "relevance-project-1": {
      "command": "npx",
      "args": ["-y", "mcp-remote", "https://mcp.relevanceai.com/"]
```

Each entry authenticates independently against its own project, so you can access tools and agents across all your projects without logging out and back in.

<Note>
  Alternatively, you can use a single connection and log out / log back in to switch projects — but the multi-connection approach above is preferred for convenience.
</Note>

## Available MCP tools

The Relevance AI MCP server exposes a set of built-in tools your AI assistant uses to read and manage your agents, tools, and workforces. This section documents the tools for working with agent system prompts.

### Agent system prompt tools

Agent system prompts can grow to hundreds or thousands of lines as you add instructions, examples, edge cases, and integration-specific rules. Loading an entire prompt into an AI assistant's context window at once becomes impractical — or impossible — once a prompt exceeds a few hundred lines. These three tools let your AI client navigate and edit prompts of any size by reading in pages, searching by pattern, and making targeted string replacements.

<Note>These tools are designed for use with AI coding assistants — Claude Code, Cursor, VS Code — not for direct API calls. The web UI remains the better option for major prompt rewrites.</Note>

The `relevance_get_agent` tool now includes a `system_prompt_line_count` field in its response. Your AI client can check this value before deciding whether to load the full prompt or use the paginated reading and search tools below.

#### relevance_read_agent_system_prompt

Reads a line-numbered window of an agent's system prompt, returning a specific range of lines rather than the full prompt. Call it multiple times with different `offset_line` values to page through a large prompt section by section.

`agent_id` identifies which agent to read from. `offset_line` sets the first line to return (default: 1). `limit_lines` controls how many lines to read in one call (default: 50). Pass `limit_lines: 0` to return only the total line count without reading any content. The tool returns the requested lines with their line numbers, the total line count in the prompt, and a `has_more` flag that indicates whether additional lines follow the returned window.

This tool is auto-approved and runs without pausing for user confirmation.

<Accordion title="Example: read lines 100–150 of a Support Agent's prompt">
  Ask your AI client: *"Show me lines 100 to 150 of my Support Agent's system prompt."*

  The AI client calls `relevance_read_agent_system_prompt` with `agent_id` set to your Support Agent's ID, `offset_line: 100`, and `limit_lines: 50`. It returns lines 100–150 with their line numbers, plus the total line count and whether there are more lines after line 150. You can then ask to continue reading from line 151, or search for a specific term using `relevance_search_agent_system_prompt`.
</Accordion>

#### relevance_search_agent_system_prompt

Searches an agent's system prompt using a regex pattern and returns matching lines with surrounding context. Use this to find all references to a specific term, integration name, or instruction without reading through the full prompt.

`agent_id` identifies which agent to search. `pattern` is a regex string matched against each line. `context` controls how many surrounding lines to include with each match (default: 2). The tool returns each matching line with its line number plus the surrounding context lines, giving you enough orientation to understand how the term is used in each location.

This tool is auto-approved and runs without pausing for user confirmation.

<Accordion title="Example: find all HubSpot references in a Sales Agent's prompt">
  Ask your AI client: *"Search my Sales Agent's system prompt for all references to 'HubSpot'."*

  The AI client calls `relevance_search_agent_system_prompt` with `pattern: "HubSpot"` and returns each matching line along with two lines of context above and below. If HubSpot appears in five different places in the prompt, you see all five matches at once, making it easy to assess whether the references are consistent and up to date.
</Accordion>

#### relevance_edit_agent_system_prompt

Replaces a specific string in an agent's system prompt and saves the result as a draft. Designed for targeted edits — changing a specific phrase, updating an integration name, or correcting a rule — without touching anything else in the prompt.

`agent_id` identifies which agent to edit. `old_string` is the exact string to find and replace — it must match the existing text exactly, including whitespace and punctuation. `new_string` is the replacement string. The tool returns confirmation that the draft was created along with the line numbers affected by the replacement.

This tool requires user approval. When your AI client calls it, your MCP client pauses and displays the proposed change — the exact text being replaced and what it will be replaced with — before applying it. Approve or reject the change before the draft is saved.

<Accordion title="Example: update an escalation phrase in a Support Agent's prompt">
  Ask your AI client: *"Replace 'escalate to tier 2' with 'escalate to senior support' in my Support Agent's system prompt."*

  The AI client calls `relevance_edit_agent_system_prompt` with `old_string: "escalate to tier 2"` and `new_string: "escalate to senior support"`. Your MCP client pauses and shows you the proposed change. Once you approve, the replacement is applied and saved as a draft. If the phrase appeared in multiple locations, all instances are replaced in a single operation.
</Accordion>

### Approval workflow

Read and search operations run automatically without requiring confirmation. Edit operations pause for explicit user approval before any change is saved.

`relevance_read_agent_system_prompt` and `relevance_search_agent_system_prompt` are auto-approved — your AI client calls them in the background as part of exploring or analyzing a prompt.

`relevance_edit_agent_system_prompt` always requires approval. Your MCP client surfaces the exact proposed change before applying it, so you can verify what is being modified before confirming. This applies even when the edit seems minor — agents handling live conversations should not be modified without review.

### Complementary to the web UI

These tools work alongside the Relevance AI web UI rather than replacing it. The web UI is better suited for major rewrites where you want to rework large sections of a prompt or restructure it significantly. The MCP tools are better for targeted refinements — updating a specific instruction, fixing a reference, or adjusting a single rule — especially in large prompts where navigating through the UI is time-consuming.

## Handling long-running agent executions

When triggering agents via MCP, you have two execution modes available depending on how long your agent takes to complete.

### Execution modes

<Tabs>
  <Tab title="Synchronous">
    The `relevance_trigger_agent` tool waits for the agent to finish and returns the result directly. It has a 120-second timeout, so use it for agents that complete quickly — single-step agents with minimal tool usage and no workforce nodes.
  </Tab>

  <Tab title="Asynchronous">
    The `relevance_trigger_agent_async` and `relevance_poll_agent_result` tools work together with no timeout limit. The trigger returns immediately with a conversation ID, which you then poll to check status and retrieve results. Use this for agents with workforce nodes, multi-step chains, external API calls, or any execution expected to exceed 2 minutes.
  </Tab>
</Tabs>

<Warning>
  If you encounter timeout errors with `relevance_trigger_agent`, switch to the async pattern. Agents with workforce nodes should always use async execution.
</Warning>

### Async execution workflow

<Steps>
  <Step title="Trigger the agent">
    Call `relevance_trigger_agent_async` with your agent parameters. This returns immediately with a `conversation_id`.
  </Step>

  <Step title="Poll for results">
    Use `relevance_poll_agent_result` with the `conversation_id` to check the execution status. Poll every 3-5 seconds until the status is `complete` or `failed`.
  </Step>

  <Step title="Retrieve the results">
    The status will be `working` or `in progress` while the agent is executing, `complete` when results are available, or `failed` if an error occurred. Once complete, the poll response contains the agent's output.
  </Step>
</Steps>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Authentication issues">
    - Make sure you have an active Relevance AI account
    - Check that you have access to the project you are trying to connect to
    - Try removing and re-adding the MCP server connection
  </Accordion>

  <Accordion title="Tools not appearing">
    - Verify that you have tools configured in your Relevance AI project
    - Check that you are authenticated to the correct project
    - Try disconnecting and reconnecting the MCP server
  </Accordion>

  <Accordion title="Connection errors">
    - Ensure you have a stable internet connection
    - Check that `https://mcp.relevanceai.com/` is accessible from your network
    - Try removing and re-adding the MCP server connection in your client
    - Try clearing the auth cache: `rm -rf ~/.mcp-auth`
  </Accordion>

  <Accordion title="Agent execution timeouts">
    Timeout errors from `relevance_trigger_agent` mean your agent exceeded the 120-second synchronous limit. Switch to `relevance_trigger_agent_async` and `relevance_poll_agent_result` instead. See [handling long-running agent executions](#handling-long-running-agent-executions) for the full workflow. Agents with workforce nodes, multi-step chains, or complex workflows should always use the async pattern.
  </Accordion>
</AccordionGroup>

## Frequently asked questions (FAQs)

<AccordionGroup>
  <Accordion title="What is MCP?">
    The Model Context Protocol (MCP) is an open standard that allows AI clients to connect to external tools and data sources. It provides a standardized way for AI assistants to access your Relevance AI workspace.
  </Accordion>

  <Accordion title="Is the MCP server free to use?">
    The MCP server itself is free. You will be billed for any Relevance AI usage (agent runs, tool executions, etc.) according to your plan.
  </Accordion>

  <Accordion title="Can I use multiple AI clients at the same time?">
    Yes. You can connect to the Relevance AI MCP server from as many clients as you like simultaneously. Each client authenticates independently.
  </Accordion>

  <Accordion title="Does authentication expire?">
    Authentication tokens may expire after a period of inactivity. If you are prompted to re-authenticate, simply follow the login flow again.
  </Accordion>

  <Accordion title="Can I restrict which tools are available via MCP?">
    The MCP server exposes the tools and agents available in the project you authenticated against. To control access, organize your tools across different projects and authenticate each connection to the appropriate project.
  </Accordion>
</AccordionGroup>
