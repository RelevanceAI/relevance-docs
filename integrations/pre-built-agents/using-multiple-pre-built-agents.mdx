# Using multiple pre-built agents

Multiple pre-built agents can work together to complete complex tasks that span across different platforms and tools. Instead of using a single agent, you orchestrate several agents to collaborate on a unified goal.

## Pre-built agents vs. custom agents in a workforce

<CardGroup cols={2}>
  <Card title="Pre-built agents" icon="store">
    Uses pre-built agents from the Marketplace designed for specific platforms

    - Ready-to-use agents (Notion, Gmail, HubSpot, etc.)
    - No setup required beyond connecting integrations
    - Optimized for common platform tasks
    - Best for cross-platform workflows
  </Card>

  <Card title="Custom agents" icon="hammer">
    Uses custom agents you build yourself for specific business needs

    - Tailored to your unique processes
    - Can include custom tools and logic
    - Requires agent building and configuration
    - Best for specialized workflows
  </Card>
</CardGroup>

<Note>
You can combine pre-built agents with your custom agents in the same workforce. Use both pre-built agents and custom agents together to create hybrid workflows that leverage the best of both. For example: use a custom research agent to gather data, then have the Gmail agent send results to your team.
</Note>

## How it works

Each pre-built agent in your workforce specializes in one platform (Notion, Gmail, Google Docs, etc.) and has access to that platform's full API capabilities. When you give a complex task that requires multiple platforms, the agents automatically:

1. **Break down the task** into platform-specific subtasks
2. **Execute in sequence** - each agent completes its part
3. **Pass information** between agents as needed
4. **Deliver the complete result** - all steps handled automatically

## Workforce example

Here's an example of multiple agents working together:

### The task

```
"Please research dinosaurs, create a presentation about them,
save the presentation to a folder called 'DinosaurFacts' and
email the slides to 'colleague@company.com'"
```

### What happens behind the scenes

This single request automatically coordinates **a mix of pre-built agents and a custom agent**:

<Note>
This workforce combines **3 pre-built agents** (Google Slides, Google Drive, Gmail) with **1 research agent** to complete the task.
</Note>

<Steps>
  <Step title="Research agent (custom)">
    Searches for relevant information about dinosaurs from knowledge sources or the web
  </Step>

  <Step title="Google Slides agent (pre-built)">
    Creates a presentation with the researched content, formats slides, adds structure
  </Step>

  <Step title="Google Drive agent (pre-built)">
    Creates the "DinosaurFacts" folder and saves the presentation to it
  </Step>

  <Step title="Gmail agent (pre-built)">
    Composes and sends an email with the presentation link to the specified recipient
  </Step>
</Steps>

**Result:** Simply describe what you need in plain English, and it all happens automatically.

## Tips for workforce tasks

<AccordionGroup>
  <Accordion title="Be explicit about the workflow" icon="list-check">
    Mention each platform and what should happen in each step to ensure the workforce understands the complete task.

    **Example:** *"Get my recent contacts from HubSpot, create a summary document in Google Docs, and email it to my team"*
  </Accordion>

  <Accordion title="Specify the order" icon="arrow-down-1-9">
    Use sequential language like "first", "then", "finally" when the order of operations matters for your workflow.

    **Example:** *"First create the Notion database, then add sample projects, finally send me a summary email"*
  </Accordion>

  <Accordion title="Include identifiers" icon="tag">
    Provide specific file names, email addresses, folder names, or collection names so agents know exactly where to work.

    **Example:** *"Save to folder 'Q1 Reports'" or "Email to team@company.com"*
  </Accordion>

  <Accordion title="Test individual steps" icon="flask">
    Before running a complex multi-agent workflow, test that each agent can handle its part independently. This helps identify issues early and ensures smoother execution.
  </Accordion>
</AccordionGroup>
