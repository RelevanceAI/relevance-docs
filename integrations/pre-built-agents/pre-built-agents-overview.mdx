# Pre-built agents

Pre-built agents understand what you want to achieve and automatically figure out the steps needed to get it done. Just describe your task in plain English, and the agent handles the technical details behind the scenes.

**Example:**
- **You say:** "Create a project tracker in Notion with tasks, owners, and due dates"
- **Agent does:** Creates databases, sets up relationships, adds fields, and configures views—automatically

## Pre-built agents vs. agents you build

<CardGroup cols={2}>
  <Card title="Pre-built agents" icon="store">
    Pre-built agents optimized for specific platforms with generic API tools

    • Ready-to-use from the Marketplace

    • Dynamically figure out API calls needed

    • Come with platform documentation built-in

    • Quick setup with no coding required

    • Best for automating work within specific tools
  </Card>

  <Card title="Agents you build" icon="hammer">
    Custom agents you build from scratch for your unique business needs

    • Define capabilities with custom tools and prompts

    • Best for specialized workflows and business processes
  </Card>
</CardGroup>

## Pre-built agents vs. integration tool steps

<CardGroup cols={2}>
  <Card title="Pre-built agents" icon="store">
    Complete agents cloned from the Marketplace that use a flexible API tool to dynamically determine what calls to make based on your natural language request

    • One agent handles many different tasks across the full platform API

    • No need to configure individual tool steps

    • Best when you want to describe goals, not steps
  </Card>

  <Card title="Integration tool steps" icon="plug">
    Individual tool steps you add to a custom agent, each configured to make a specific API call with specific parameters

    • Precise, repeatable actions in structured workflows

    • Full control over each API call

    • Best for predictable, production workflows
  </Card>
</CardGroup>

## How they work

<Steps>
  <Step title="Describe what you want">
    Tell the agent your goal in everyday language—just like delegating to a colleague
  </Step>

  <Step title="Agent figures it out">
    The agent determines what API calls to make and in what order
  </Step>

  <Step title="Task completed">
    Your request is fulfilled and you receive confirmation with results
  </Step>
</Steps>

The agent uses its understanding of the tool's API to dynamically construct the right sequence of actions. You don't need to know anything about APIs, authentication, or technical implementation.

## Real-world examples

<AccordionGroup>
  <Accordion title="Notion: build a complete task management system">
    **Request:** "Create a project management system with linked databases for projects and tasks"

    **What the agent does:**
    - Creates a Projects database with fields for name, status, and owner
    - Creates a Tasks database with title, assignee, due date, and status
    - Sets up relationships between projects and tasks
    - Configures views and filters
    - All automatically
  </Accordion>

  <Accordion title="Webflow: create a contact form">
    **Request:** "Create a contact form on my site with fields for name, email, phone, and message"

    **What the agent does:**
    - Creates a form collection with appropriate field types
    - Configures validation rules
    - Sets up the form structure
    - Returns the form ID and details
  </Accordion>

  <Accordion title="HubSpot: segment and tag contacts">
    **Request:** "Find all contacts who haven't been contacted in 30 days and add them to a re-engagement list"

    **What the agent does:**
    - Searches contacts with date filters
    - Creates or identifies the re-engagement list
    - Adds matching contacts to the list
    - Provides a summary of contacts added
  </Accordion>
</AccordionGroup>
