## Available Models

OpenAI offers multiple model variants optimized for different use cases. All models support multimodal inputs (images and files), function calling, and configurable reasoning levels.

### GPT 5.4

The flagship model with a 1M+ context window and state-of-the-art capabilities across coding, computer use, and tool search.

**Key Specifications:**
- **Context window:** 1,050,000 tokens (1M+)
- **Max output tokens:** 128,000 tokens
- **Reasoning levels:** Supports none, low, medium, high, and xhigh
- **Multimodal support:** Images and files (.pdf, .jpg, .jpeg, .gif, .webp, .png)
- **Function calling:** Full support

**Pricing:**
- Input: 0.00125 credits per 1K tokens
- Output: 0.0075 credits per 1K tokens

**Best for:**
- Complex coding tasks requiring large context
- Multi-step agentic workflows
- Tasks requiring deep reasoning
- Processing large documents

### GPT 5.4 Pro

The most powerful variant that uses additional compute to think harder on complex problems.

**Key Specifications:**
- **Context window:** 1,050,000 tokens (1M+)
- **Max output tokens:** 128,000 tokens
- **Reasoning levels:** Supports medium, high, and xhigh (minimum medium)
- **Multimodal support:** Images and files (.pdf, .jpg, .jpeg, .gif, .webp, .png)
- **Function calling:** Full support

**Pricing:**
- Input: 0.015 credits per 1K tokens
- Output: 0.09 credits per 1K tokens

<Warning>
GPT 5.4 Pro is significantly more expensive than GPT 5.4. Use only when additional reasoning capability justifies the cost.
</Warning>

**Best for:**
- Extremely complex reasoning tasks
- Advanced mathematical or scientific problems
- Tasks where accuracy is critical and cost is secondary

### Choosing the Right Model

<AccordionGroup>
  <Accordion title="When to use GPT 5.4">
    Choose GPT 5.4 for most production workloads where you need:
    - Strong performance across general tasks
    - Large context windows for document processing
    - Balanced cost and capability
    - Flexibility to disable reasoning when not needed
  </Accordion>
  <Accordion title="When to use GPT 5.4 Pro">
    Choose GPT 5.4 Pro only when you need:
    - Maximum reasoning capability on hard problems
    - Highest possible accuracy regardless of cost
    - Complex multi-step problem solving
    - Advanced mathematical or scientific computation
  </Accordion>
</AccordionGroup>

<Note>
You can configure reasoning levels for your agent in the [Language Model settings](/build/agents/build-your-agent/agent-settings/language-model). Higher reasoning levels use more output tokens and increase cost, but improve performance on complex problems.
</Note>

## Understanding pricing & implicit caching

OpenAI models automatically cache repeated input tokens (like agent instructions and tool definitions) in multi-turn conversations, charging them at **90% less** than uncached tokens. This happens automatically with no code changes required.

### Pricing example: GPT-4o

| Token Type | Cost per 1,000 tokens |
|------------|----------------------|
| **Uncached input tokens** | 0.63 credits |
| **Cached input tokens** | 0.063 credits |
| **Output tokens** | 5 credits |

**Example:** An agent with 19,000 tokens of instructions makes two calls:
- **First call:** 19,000 input (uncached) + 2,000 output = 21.97 credits
- **Second call:** 6,000 input (uncached) + 13,000 input (cached) + 1,500 output = 12.10 credits
- **Without caching**, the second call would cost 19.47 credits
- **Savings: 38% reduction** on the second call

### Why UI numbers may look different

<Callout icon="info-circle">
The task credit breakdown displays **uncached input tokens only**. Cached tokens are charged separately at the lower rate but not shown in the breakdown. The **total cost is accurate** and includes both.
</Callout>

If you see "6,000 input tokens" in the UI, there may be additional cached tokens charged at 90% less. Your actual costs are **lower than expected** because of automatic caching.

<Tip>
**No markup policy:** Relevance AI passes through OpenAI's exact pricing, including all caching discounts. We do not add any markup to Vendor Credits.
</Tip>

For technical details about OpenAI's prompt caching implementation, see [OpenAI's Prompt Caching Documentation](https://platform.openai.com/docs/guides/prompt-caching).
