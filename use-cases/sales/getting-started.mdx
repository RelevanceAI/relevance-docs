  ## Step 2 — Connect the tools it needs

  Sales Agents work best when they can read from and write to the systems your team already uses.

  - **CRM** — [HubSpot](/integrations/popular-integrations/hubspot) or [Salesforce](/integrations/popular-integrations/salesforce) for account data and activity logging
  - **Email** — [Gmail](/integrations/popular-integrations/gmail) or Outlook for sending outreach
  - **Prospecting** — [Apollo](/integrations/popular-integrations/apollo) or ZoomInfo for contact and firmographic data

  Full list at [Integrations](/integrations/introduction). Start with your CRM and one outreach channel; add the rest as the workflow grows.

  ## Sales-specific pointers

  A few patterns that pay off when you're building sales Agents this way:

  - **Version your prompts in git.** Programmatic GTM lets you pull the system prompt locally, edit it, and push it back. Treat Agent prompts like any other source file — diff them, review them, roll them back.
  - **Generate triggers from CRM events.** Wire HubSpot or Salesforce webhooks into Agent triggers programmatically so research, enrichment, and outreach fire on real pipeline signals — new lead, stage change, missed reply.
  - **Compose workforces in code.** Sales workflows usually chain — research → enrich → write outreach → send. Define the workforce in your editor instead of dragging nodes. Easier to reproduce across environments.
  - **Run evals on real accounts.** Before scaling an Agent across thousands of leads, build an [eval test set](/build/agents/build-your-agent/evals) of representative accounts and measure output quality every time you change the prompt.
  - **Bulk-run from the command line.** Trigger an Agent across a list of prospects via the API or MCP rather than the UI. Same outcome, scriptable.

  ## SuperGTM works alongside

  Anything you build in Relevance — including Agents you cloned and customized programmatically — can run inside **[SuperGTM](/get-started/chat/super-gtm)** in chat mode. Build the Agent the way that suits you (code), and your sales team can invoke it conversationally without leaving chat.
