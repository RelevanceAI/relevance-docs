  ## Connect the tools it needs

  Sales Agents work best when they can read from and write to the systems your team already uses.

  - **CRM** — [HubSpot](/integrations/popular-integrations/hubspot) or [Salesforce](/integrations/popular-integrations/salesforce) for account data and activity logging
  - **Email** — [Gmail](/integrations/popular-integrations/gmail) or Outlook for sending outreach
  - **Prospecting** — [Apollo](/integrations/popular-integrations/apollo) or ZoomInfo for contact and firmographic data

  Full list at [Integrations](/integrations/introduction). Start with your CRM and one outreach channel; add the rest as the workflow grows.

  ## Drop into code when you need to

  Programmatic GTM covers the building. Two escape hatches for when the canned Tool steps fall short:

  - **Custom Python and JS code blocks** live inside Tools. Parse a weird API response, apply business logic, transform data — drop a code block in and write the logic directly.
  - **The JS SDK** lets you call Agents and Tools from your own Node.js app. Useful when you're embedding Relevance into a TypeScript backend or calling an Agent from a webhook handler.

## The four levels in sales

How the [autonomy ladder](/use-cases/guides) plays out across your pipeline:

<CardGroup cols={2}>
  <Card title="L1 — Assisted" icon="hand">

  </Card>
  <Card title="L2 — Copilot" icon="map" href="/build/agents/give-your-agent-tasks/approval-mode">
    A rep asks for 12 personalized LinkedIn messages — the Agent drafts all 12, the rep approves and sends.
  </Card>
  <Card title="L3 — Autopilot" icon="bolt" href="/build/workforces/create-a-workforce">
