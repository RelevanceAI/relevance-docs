This endpoint returns time-series data showing your organization's concurrency usage over a specified date range. Use it to monitor usage patterns, identify peak periods, and plan capacity.

## Endpoint

```
GET https://api-{region}.stack.tryrelevance.com/latest/analytics/get_concurrency_time_series
```

Replace `{region}` with your organization's region (e.g., `f1`, `d1`, `e1`).

## Requirements

<Warning>
This endpoint requires **admin permissions** and is available on **Team plans and above**.
</Warning>

- **Admin permission**: You must have admin-level access to your organization. See [Role-based access controls](/enterprise/rbac).
- **Team plan or higher**: Available on Team, Business, and Enterprise plans. [View pricing](https://relevanceai.com/pricing).

## Parameters

<ParamField query="start_date" type="string" required>
  Start of the date range in ISO 8601 format.
  
  Example: `2024-01-01T00:00:00Z`
</ParamField>

<ParamField query="end_date" type="string" required>
  End of the date range in ISO 8601 format.
  
  Example: `2024-01-31T23:59:59Z`
</ParamField>

<ParamField query="granularity" type="string" default="hourly">
  Time bucket size for data aggregation.
  
  **Options:**
  - `15min` - 15-minute intervals
  - `hourly` - 1-hour intervals (default)
</ParamField>

<ParamField query="aggregation" type="string" default="max">
  Aggregation method for concurrency values within each time bucket.
  
  **Options:**
  - `max` - Maximum concurrency in the time bucket (default)
  - `avg` - Average concurrency in the time bucket
</ParamField>

<ParamField query="timezone" type="string" default="UTC">
  IANA timezone identifier for date range interpretation.
  
  Examples: `America/New_York`, `Europe/London`, `Asia/Tokyo`
  
  Default: `UTC`
</ParamField>

## Response

The endpoint returns a JSON object containing time-series data and metadata about your organization's concurrency limit.

<ResponseField name="time_series" type="array">
  Array of data points, each containing a timestamp and concurrency value.
  
  <Expandable title="time_series items">
    <ResponseField name="timestamp" type="string">
      ISO 8601 timestamp for the data point.
    </ResponseField>
    
    <ResponseField name="value" type="number">
      Concurrency value at this timestamp, based on the specified aggregation method.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="metadata" type="object">
  Additional information about the query and organization limits.
  
  <Expandable title="metadata properties">
    <ResponseField name="concurrency_limit" type="number">
      Your organization's effective concurrency limit based on your subscription tier.
    </ResponseField>
    
    <ResponseField name="start_date" type="string">
      Start date of the queried range (echoed from request).
    </ResponseField>
    
    <ResponseField name="end_date" type="string">
      End date of the queried range (echoed from request).
    </ResponseField>
    
    <ResponseField name="granularity" type="string">
      Time bucket size used for aggregation (echoed from request).
    </ResponseField>
    
    <ResponseField name="aggregation" type="string">
      Aggregation method used (echoed from request).
    </ResponseField>
  </Expandable>
</ResponseField>

## Understanding concurrency

Concurrency refers to the number of tasks that can be executed simultaneously by your agents. Each subscription tier provides different concurrency capacity:

- Higher concurrency allows more parallel agent executions
- When capacity is reached, additional tasks are queued
- Enterprise plans can include custom concurrency limits

<Tip>
Use the `concurrency_limit` value in the response metadata to compare your usage against your tier's limit.
</Tip>
