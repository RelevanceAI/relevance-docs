The Analytics API provides programmatic access to your organization's usage data and metrics. These endpoints allow you to retrieve time-series data for monitoring, reporting, and capacity planning.

## Authentication

All Analytics API endpoints require authentication using your organization's API key with admin permissions.

```bash
'Authorization': 'Bearer <your_api_key>'
```

<Warning>
Analytics API endpoints require **admin permissions**. Users without admin access will receive a 403 Forbidden error.
</Warning>

## Base URL

Analytics API endpoints are available at your region-specific base URL:

```
https://api-{region}.stack.tryrelevance.com/latest/analytics
```

Replace `{region}` with your organization's region (e.g., `f1`, `d1`, `e1`).

## Common parameters

Many Analytics API endpoints share common parameters:

| Parameter | Type | Description |
|-----------|------|-------------|
| `start_date` | string | Start of date range (ISO 8601 format) |
| `end_date` | string | End of date range (ISO 8601 format) |
| `timezone` | string | IANA timezone identifier (default: UTC) |
| `granularity` | string | Time bucket size for aggregation |
