<Update label="June 29, 2026" description="Eval action billing, cost breakdowns, and Workforce sub-agent/tool support">

## Eval action billing, cost breakdowns, and Workforce sub-agent/tool support

Three changes to Evals and Workforces are now live for all users.

Eval Checks now consume 1 action per check run — this applies to all check types, including LLM as Judge and Tool Usage checks. Each completed Eval run now renders a cost breakdown panel that lists action consumption per individual check, so users can see exactly how a run's total action count is composed. Separately, Workforces now support sub-agent and tool invocations within their runs; previously this tracking and invocation layer was only wired up for individual Agent runs. Workforce runs that include sub-agent calls or tool usage will now correctly process and record those interactions in the same way individual Agent runs do.

For example, a Workforce that orchestrates multiple agents — each calling external tools — can now be fully evaluated with accurate action counts attributed per check.

To view cost breakdowns, open any completed Eval run; the breakdown panel appears alongside the results. Workforce sub-agent and tool support is enabled automatically with no configuration required. See the [Evals documentation](/evals) for full details.

</Update>

<Update label="May 29, 2026" description="Timeline view added to Tasks page">

## Timeline view added to Tasks page

![Tasks page hero showing the Approvals, Escalated, and Errors timeline charts stacked on the left and a task list with mixed statuses on the right](/images/changelog/changelog-taskops-timeline.png)

The **Approvals**, **Escalated**, and **Errors** tabs on the Tasks page now show a timeline chart of task counts per day.

Each tab is colored to match its sentiment — Approvals yellow, Escalated orange, Errors red — and the bars are day-bucketed in your local timezone. The chart honors the date range and Agent filters at the top of the page, so the bars always match the task list below. Any project member who can already see a tab can also see its chart; no admin role required.

Find the chart at the top of each tab in **Tasks → Approvals / Escalated / Errors**.

</Update>

<Update label="May 21, 2026" description="Outreach-only reply mode for Microsoft Teams agents">

## Outreach-only reply mode for Microsoft Teams agents

![Microsoft Teams trigger settings showing the new Outreach replies only toggle enabled](/images/changelog/changelog-teams-outreach.png)

Microsoft Teams triggers now support an **Outreach replies only** mode that limits responses to direct replies on the Agent's own outbound DMs.

The trigger distinguishes between inbound replies (replies to messages your Agent sent) and channel mentions. In outreach-only mode, only inbound replies route to the Agent — channel chatter doesn't trigger a run.

Configure the toggle on your Microsoft Teams trigger in **Agent → Triggers → Microsoft Teams**.

</Update>
