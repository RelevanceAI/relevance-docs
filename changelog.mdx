<Update label="May 13, 2026" description="MCP access for the Member role">

## MCP access for the Member role

The Relevance MCP is now available to users with the Member role in RBAC-enabled organizations. Previously, this required the Editor role or above. Requests use the user's own auth, so Members only see and operate on resources they already have access to.

</Update>

<Update label="May 1, 2026" description="3 updates">

## Confluence Knowledge Sync

![](/images/changelog/changelog-confluence.webp)

Knowledge tables can now sync from Confluence. Pages are polled every 5 minutes and changes flow back into the connected table — same model as the existing Google Drive, Notion, and SharePoint sync.

Connect in **Knowledge → Add Source → Confluence** and authenticate your workspace.

## Project-level public Agent controls

![](/images/changelog/changelog-public-access.webp)

Admins can now disable public cloning and embedding for every Agent, Workforce, and Tool in a project from one place. Toggling these off at the project level overrides individual asset settings and stops users from re-enabling them.

Manage in **Project Settings → Manage Access**.

## Relevance AI Android app

![](/images/changelog/changelog-google-play-2.webp)

The Android app is live on the Google Play Store. Trigger Agents, view runs, and receive notifications from your phone.

[Get it on Google Play](https://play.google.com/store/apps/details?id=com.relevanceai.android).

</Update>

<Update label="March 13, 2026" description="3 updates">

## Agent Performance Observability

![](/images/changelog/changelog-observability.webp)

Production monitoring for Agents using LLM judges. Define evaluation criteria (summary quality, hallucination detection, workflow execution), pick a sample rate, and watch scores trend over time. Drill into failed runs to see what went wrong.

- **Sampling**: 10%, 50%, or 100% of conversations
- **Dashboard**: overall scores, per-evaluator breakdowns, time-series trends
- **Investigation**: filter and inspect individual failed runs

Find it under your Agent's **Monitor → Test Suites**.

<Note>Enterprise plan with eval access. Talk to your Relevance AI rep if you don't see it.</Note>

## Pause, cancel, and resume Agent triggers

![](/images/changelog/changelog-pause-agent-triggers.webp)

Agent triggers now pause without being deleted — same controls that exist on Workforces. Toggle individual triggers, or pause every trigger on an Agent at once. Paused triggers keep their configuration.

Manage from **Agent → Triggers**.

## Chat people search

![](/images/changelog/changelog-chat-search.webp)

Chat now includes people search powered by [Exa](https://exa.ai). Ask Chat to find people by role, location, or skills and it returns profiles with current roles and companies.

Example: *"Find me product designers in Sydney with SaaS experience."*

</Update>

<Update label="March 10, 2026" description="Org concurrency timeseries">

![](/images/changelog/changelog-concurrency.webp)

The Analytics dashboard now plots your org's concurrent operations as a timeseries over the last 7 days, with your plan limit drawn on the chart so you can see how close you've been running to the ceiling.

<Note>Requires Organization Admin.</Note>

</Update>

<Update label="March 9, 2026" description="2 updates">

## Microsoft Teams integration

![](/images/changelog/changelog-teams.webp)

The Relevance AI app is live in the Microsoft Teams App Store. Trigger Agents from Teams channels and chats, and receive run results back in the same thread.

## Project invitation email tracking

Project invitation emails now show delivery, bounce, and open status — and let you resend invites that bounced or got missed. No more guessing whether a teammate's invite landed.

</Update>

<Update label="February 24, 2026" description="Teams calendar trigger">

![](/images/changelog/changelog-teams-trigger.webp)

Agents can now be triggered by Microsoft Teams calendar events — meeting start, end, etc. Parity with the existing Google Calendar trigger, so Microsoft 365 teams can wire up the same meeting-prep, follow-up, and summary workflows.

<Update label="February 17, 2026" description="Enterprise analytics: usage and error tracking">

![](/images/changelog/changelog-analytics.webp)

Two new dashboards on the project Analytics page:

- **Usage Overview** — which Agents, Tools, and Workforces consume the most actions and credits
- **Error Timeline** — error counts over time, filterable by Agents / Tools / Workforces, with the underlying error message visible per data point

</Update>

<Update label="February 11, 2026" description="Claude Opus 4.6">

![](/images/changelog/68faec2d3ffbd34e9c9295b9-changelog-claude.webp)

Anthropic's flagship Opus 4.6 is now available in the model picker.

- **Context window**: 1M tokens (both the base entry and the "1M context" entry — the second one exists for pricing reasons, with separate tiers for ≤200K and >200K input)
- **Adaptive thinking**: supported by default
- **Image inputs**: jpg, jpeg, gif, webp, png

Select **Claude Opus 4.6** from the model menu.

</Update>

<Update label="February 9, 2026" description="2 updates">

## Bulk run editing

![](/images/changelog/698a5fc1b1dc4b278cb242b7-changelog-edit-bulk-runs.webp)

You can now edit a bulk run after it's been created — change daily task volume, change time zone, change other parameters in the bulk run modal — instead of cancelling and starting over.

Open your Agent, select the active bulk run, and click **Edit**.

## Evals: Agent testing and validation

![](/images/changelog/698a6f9d99180607d34d642c-changelog-evals.webp)

Structured testing for Agents. Create test suites with multiple scenarios, generate user inputs, and evaluate runs with LLM-based judges against criteria you define. Track scores across test runs so regressions show up.

Tool-call simulation (run an Agent through a flow without firing real Tool side effects) is coming next.

Find it under your Agent's **Monitor → Test Suites**.

<Note>Enterprise only — talk to your Relevance AI rep for access.</Note>

</Update>

<Update label="February 1, 2026" description="AWS Bedrock Guardrails for BYO Anthropic">

![](/images/changelog/69800602a1c89e5e19ffd9f5-changelog-aws-bedrock-and-anthropic.webp)

If you're running Anthropic models through your own AWS Bedrock account (BYO), Relevance now forwards a guardrail identifier and version on every Bedrock invoke. The actual filters — prompt-injection blocking, PII masking, custom topic restrictions, profanity controls — are configured AWS-side in your Bedrock console; Relevance passes the ARN through.

Set the guardrail ID/version in your project's Anthropic integration settings.

[Setup guide →](https://relevanceai.com/docs/integrations/llm-integrations/anthropic#aws-bedrock-guardrails-optional)

</Update>

<Update label="January 22, 2026" description="OpenTelemetry event streaming">

![](/images/changelog/6972e080d65aad52eeebeb9b-changelog-opentelemetry-event-streaming.webp)

Audit logs and execution traces can now be streamed to a customer-controlled destination in OpenTelemetry format (OTel spans + logs). Plug it into your existing observability stack — Datadog, Honeycomb, Grafana, whatever — and run governance and compliance queries alongside the rest of your telemetry.

<Note>Enterprise with RBAC enabled. RBAC is rolling out progressively — talk to your Relevance AI rep if you don't see it.</Note>

</Update>

<Update label="January 20, 2026" description="Workforce trigger controls">

![](/images/changelog/69700d9eac32ec1e2b9919d0-changelog-pause-workforce-triggers.webp)

Workforce triggers now support pause, cancel, and resume — both per-trigger and at the whole-Workforce level. Pause an entire Workforce during maintenance, then resume it without recreating anything.

Manage from the Workforce dashboard, or use the **Pause all triggers** button in the top menu.

</Update>

<Update label="January 19, 2026" description="2 updates">

## Slack Tool approvals

![](/images/changelog/696f13a0bd3782afb55800f7-changelog-slack-tool-approvals.webp)

Approve or reject pending Tool calls from your Agents directly in Slack, without switching contexts. The notification shows the Tool name and parameters; one click sends the approval back.

Enable per Slack trigger: **Triggers → ⋯ → Edit trigger → Advanced settings → Tool approval from Slack → Yes**.

## Plan & Billing page filters

![](/images/changelog/696f107a552f3dd900a5d66d-changelog-spend-filters.webp)

The Plan & Billing page now supports filtering credit usage by date range and by project, with side-by-side comparison and export. Useful for chargeback reporting and spotting which project is driving spend.

**Profile → Manage organization → Plan and billing**.

</Update>

<Update label="January 18, 2026" description="Test Phone Agents in your browser">

![](/images/changelog/696d6f6c9e49abe25688ecf1-changelog-phone-testing.webp)

Validate a Phone Agent's prompt, voice, and conversation flow from the dashboard without placing a real call. Microphone permission required.

On the Phone Agent's build page, hit **Test in Browser** under **Call Agent**.

</Update>

<Update label="January 4, 2026" description="2 updates">

## Meeting Agent realtime commands

![](/images/changelog/695a95a2a931a05c425b7b72-changelog-meeting-bot-real-time-commands.webp)

Type a trigger word in the meeting chat and your Meeting Agent acts on it mid-meeting — run a search, pause or resume the recording, build slides, post to Notion or Slack — without breaking the conversation.

Build a new Agent, add the **Relevance AI Meeting Bot** trigger, attach the Tools you want (e.g. Pause Recording, Resume Recording, Google Search, Slide Builder) and define each Tool's trigger word.

## Slide reordering in Slide Builder

![](/images/changelog/695a8b2e572345c02f6ae362-changelog-slide-reordering.webp)

Drag-and-drop slide reordering in Slide Builder (Chat). Hover a slide thumbnail and drag it to its new position.

</Update>
