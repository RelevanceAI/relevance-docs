Confluence can be added as an integrated Knowledge source via the Agent Builder for your Agents to read pages and use them as Knowledge.

You can select specific Confluence pages or spaces for your Agent to read as Knowledge sources. When the source pages change, your Knowledge table updates automatically, so your Agents always reference the latest documentation.

<Note>Confluence is also available as an integration for Tool steps. [Learn more about the Confluence integration](/integrations/popular-integrations/confluence).</Note>

## Set up Confluence as a Knowledge source

{/* TODO: Supademo recording of the Confluence knowledge setup */}

To set up Confluence as a Knowledge source, first open an Agent and head to the Build screen. Then...

1. Click 'Knowledge'
2. Click 'Add Confluence'
3. Select or connect the Confluence account you want to access
4. Select the Confluence pages or spaces you wish to have the Agent read
5. Once you've selected what you want to add as Knowledge, click 'Create'
6. Choose how you want the Agent to use this Knowledge on the next screen

## Frequently asked questions (FAQs)

<AccordionGroup>
  <Accordion title="Can I add a whole Confluence space, or only individual pages?">
    You can select individual pages or an entire space. Selecting a space adds its pages as Knowledge.
  </Accordion>
  <Accordion title="Do changes in Confluence sync back to Relevance AI?">
    Yes. When the content of a connected Confluence page changes, the Knowledge table updates automatically — you don't need to re-add the page.
  </Accordion>
</AccordionGroup>
