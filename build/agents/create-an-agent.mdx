## Option 1: Invent an Agent

The fastest way to get started. Describe what you need and we'll generate a working Agent for you.

![Select Invent to describe what you want and let AI build your Agent.](/images/quickstart-invent-agent.png)

1. Click **Agents** in the left sidebar
2. Click **+ Create Agent**
3. Select **Invent**
4. Describe what you want your Agent to do — be specific about the task, the output you expect, and any integrations it should use. For example: *"Create an agent that researches a company before a sales call. It should use web search to find recent news, funding, and key contacts, then output a one-page brief with company overview, decision makers, and talking points."*
5. Review the generated Agent — Inventor will set up the prompt and suggest Tools
6. Connect any integrations your Tools need (e.g. Gmail, HubSpot, LinkedIn) in the Tools section of the Agent builder, or [add integrations](/integrations/add-integrations) from the sidebar

<Tip>
Best when you know what you want and can describe it clearly. Great for getting a concept off the ground quickly — Inventor handles the setup, you make the tweaks.
</Tip>

## Option 2: Clone from Marketplace

Start with an Agent that's already been built and tested by domain experts.

![Browse the Marketplace to find pre-built Agents for common use cases.](/images/create-agent-marketplace.png)

1. Go to [Marketplace](https://marketplace.relevanceai.com/) or click **Marketplace** in the left sidebar
2. Browse or search for an Agent that matches your use case
3. Click on the Agent to view its details
4. Click **Clone** to add it to your workspace
5. A setup popup will walk you through connecting any integrations the Agent needs — follow the prompts to link your accounts
6. Review the prompt, Tools, and settings — adjust anything you need

<Tip>
Great when you have a common use case — like lead qualification, meeting prep, or customer support — and want a proven starting point. Clone it, tweak it, ship it.
</Tip>

## Option 3: Build from scratch

Full control from the ground up. Best if you have a specific workflow in mind.

![The Agent builder — configure your prompt, Tools, and Knowledge from scratch.](/images/build-platform-overview.png)

1. Click **Agents** in the left sidebar
2. Click **+ Create Agent**
3. Select **Start from scratch**
4. Write a [prompt](/build/agents/build-your-agent/prompt) — tell the Agent who it is, what it does, and how it should behave
5. Add [Tools](/build/agents/build-your-agent/tools) — give it the actions it needs (search the web, send emails, update your CRM, etc.)
6. Optionally add [Knowledge](/build/knowledge/create-knowledge) — upload docs or connect data sources for context

## Using Inventor v3 while building

Once you've created an Agent using any of the three methods above, you can continue building and refining it with **Inventor v3** — a conversational assistant that stays with you throughout the entire build process.

### What is Inventor v3?

Inventor v3 is a persistent sidebar in the Agent builder that lets you configure your Agent using natural language instead of navigating through menus and settings. Think of it as a building partner that understands what you're trying to accomplish and can make changes for you.

Unlike the one-time **Invent** feature you use at creation, Inventor v3 remains available as you build. You can ask it to adjust your prompt, add tools, configure triggers, update knowledge sources, or modify any other Agent setting — all through conversation.

<Note>
Inventor v3 is currently behind the `InventV3` feature flag. Contact your workspace admin or Relevance AI support to enable it.
</Note>

### How it works

Inventor v3 has access to approximately 50 specialized tools that cover every aspect of Agent configuration:

- **Prompt and behavior** — update instructions, adjust tone, refine responses
- **Tools** — add, remove, or configure the actions your Agent can take
- **Knowledge** — create tables, add data sources, manage context
- **Triggers** — set up schedules, webhooks, or integration-based automation
- **Variables** — define dynamic inputs your Agent uses
- **Advanced settings** — configure memory, escalations, model selection, and more

Simply describe what you want to change, and Inventor v3 handles the implementation. For example:

- *"Add a tool that sends a Slack message when the task is complete"*
- *"Update the prompt to be more formal and include a greeting"*
- *"Create a knowledge table for customer data with columns for name, email, and company"*
- *"Set up a daily trigger at 9 AM to run this Agent"*

### When to use Inventor v3

Inventor v3 is useful when you:

- Want to make changes quickly without clicking through multiple settings
- Need help understanding how to configure a specific feature
- Prefer describing what you want over manually implementing it
- Are learning the platform and want guidance as you build

You can switch between using Inventor v3 and manually configuring settings at any time — they work together, not as replacements for each other.

### Difference from the Invent feature

The **Invent** feature (used during Agent creation) generates an initial Agent blueprint based on your description. It's a one-time setup step.

**Inventor v3** is an ongoing assistant that stays available after creation. It helps you iterate, refine, and expand your Agent as you build. Use Invent to get started, then use Inventor v3 to continue building.

## What's next?

Once you've created an Agent, explore the rest of the build guides to refine it:

- [Prompt](/build/agents/build-your-agent/prompt) — write instructions that define your Agent's behavior
- [Tools](/build/agents/build-your-agent/tools) — give your Agent the actions it needs
- [Triggers](/build/agents/build-your-agent/triggers) — run your Agent automatically on a schedule, from a webhook, or from an integration
- [Escalations](/build/agents/build-your-agent/escalations) — define when your Agent should loop in a human
- [Memory](/build/agents/build-your-agent/memory) — configure how your Agent retains context across conversations

You can configure all of these manually or use Inventor v3 to make changes conversationally as you build.
