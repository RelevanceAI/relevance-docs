You can delete agents from the Relevance AI interface or from within Inventor. Deleting an agent is permanent and cannot be undone — it also removes the agent's syncs and scheduled triggers.

## Delete via the UI

1. Click **Agents** in the sidebar
2. Click the three-dot menu next to the agent you want to delete, then select **Delete**
3. Confirm the deletion in the dialog that appears

## Delete via Inventor

You can ask Inventor to delete an agent during a conversation.

1. Open Inventor from the sidebar
2. Ask Inventor to delete an agent, for example: "Delete this agent" or "Delete the [agent name] agent"
3. Inventor will prompt you to confirm before proceeding — confirmation is always required for deletions, even when auto-approve is enabled for other actions

<Note>
If you delete the agent you are currently editing in Inventor, you will be automatically redirected away from that agent's view.
</Note>

## Restore a previous agent version

Inventor can roll an existing agent back to an earlier version. This restores a previous *version* of an agent — it does not recover a deleted agent.

1. Open Inventor from the sidebar
2. Ask Inventor to restore an earlier version, for example: "Restore the previous version of this agent"
3. Inventor copies the chosen version into the current draft. The live (published) version is unchanged until you publish the draft.
4. Inventor will prompt you to confirm — confirmation is always required for restore actions, even when auto-approve is enabled

## Frequently asked questions (FAQs)

<AccordionGroup>
    <Accordion title="What happens if my deleted agent is clonable?">If your deleted agent is clonable, it will no longer be available for cloning. People who have already cloned it will still be able to use their copy.</Accordion>
    <Accordion title="I accidentally deleted an agent, can I recover it?">No. Deleting an agent is permanent and cannot be undone. Inventor can restore a previous *version* of an existing agent, but it cannot recover one that has been deleted — so confirm carefully before deleting.</Accordion>
</AccordionGroup>
