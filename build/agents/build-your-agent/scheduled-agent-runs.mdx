# Scheduled agent runs

Scheduled runs let you trigger your agents on a recurring basis. This guide shows you how to set them up using a cron-style schedule.

## Setting Up Your First Schedule

Navigate to your agent's settings and find the triggers section.

<Tip>
Here are the steps to add a scheduled trigger:

1. Click on the "Add trigger" button
2. Select "Schedule" from the trigger type list
3. Choose either a preset interval or enter a custom cron expression
4. Save your changes

Your agent will now run automatically based on the schedule you configured.
</Tip>

## Cron Expression Format

Standard cron format applies. For example:

```
0 9 * * 1-5
```

This runs the trigger at 9:00 AM on weekdays.

You can find more examples in our [scheduled triggers guide](/build/agents/agent-triggers/scheduled-triggers-guide).

## Memory Behaviour Across Runs

Each scheduled run starts with a fresh memory state by default. If you want to preserve memory across runs, you can use [memory snapshots](/build/agents/build-your-agent/agent-memory-snapshots) which persist for 7 days.

## Best Practices

- Use UTC for all schedules to avoid timezone confusion
- Avoid scheduling during deployment windows
- Monitor execution logs regularly
- Implement robust error handling to seamlessly recover from failures

## FAQ

### What happens if a scheduled run fails?

Failed runs are retried up to 3 times automatically.

### Can I pause a schedule without deleting it?

Yes, you can disable triggers from the triggers panel.

### How do I supercharge my scheduled workflows?

Combine scheduled runs with our powerful workforce features to leverage multi-agent coordination.
