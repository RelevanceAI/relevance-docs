## Evals API

The Evals API enables developers to programmatically access evaluation run data for their AI Agents. This endpoint provides comprehensive filtering, pagination, and sorting capabilities for retrieving evaluation results.

<Warning>
The Relevance AI API is officially supported for triggering Agents and Tools only. All other API usage is currently unsupported.
</Warning>

### Key Features

- **Programmatic Access**: Retrieve evaluation run data through HTTP requests
- **Advanced Filtering**: Filter by batch ID, evaluator results, date ranges, status, and more
- **Pagination Support**: Handle large result sets efficiently with cursor-based pagination
- **Flexible Sorting**: Sort results by any field in ascending or descending order
- **Integration Ready**: Works with any programming language capable of making HTTP requests

### How the Evals API Works

The Evals API exposes an endpoint that accepts HTTP POST requests with filtering and pagination parameters. When you send a request with your agent ID and optional filters, the API returns a paginated list of evaluation runs matching your criteria.

<Note>
You must have access to the specified agent to retrieve its evaluation data. Authentication is required for all API requests.
</Note>

### Authentication

All API requests require authentication using your Relevance AI API key. Include your API key in the `Authorization` header of each request.

To generate an API key:

1. Navigate to your project settings
2. Go to the API Keys section
3. Click "Generate API Key"
4. Store your key securely - you won't be able to view it again

<Warning>
Store your API key securely and use environment variables rather than hardcoding it in your application. Never commit API keys to version control.
</Warning>

## API Reference

### List Evaluation Runs

Retrieve a paginated list of evaluation runs for a specific agent.

**Endpoint:**
```http
POST https://api-f1db6c.stack.tryrelevance.com/latest/evals/agents/:agent_id/runs
```

<Info>
The endpoint URL differs based on your organization's region. The example above shows the AU region endpoint. Check your project settings for the correct endpoint URL for your region.
</Info>

**Path Parameters:**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `agent_id` | string | Yes | The unique identifier of the agent |

**Request Body:**

All request body parameters are optional. If no filters are provided, the endpoint returns all evaluation runs for the agent.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `batch_id` | string | No | Filter runs by specific batch identifier |
| `evaluator_results` | string[] | No | Filter by evaluator outcomes (e.g., `["pass", "fail"]`) |
| `date_range` | object | No | Filter by date range with `from` and `to` timestamps (ISO 8601 format) |
| `status` | string | No | Filter by run status (`completed`, `running`, `failed`) |
| `scenario` | string | No | Filter by test scenario name |
| `agent_version` | string | No | Filter by specific agent version |
| `limit` | number | No | Number of results per page (default: 20, max: 100) |
| `offset` | number | No | Number of results to skip for pagination |
| `cursor` | string | No | Cursor for pagination (alternative to offset) |
| `sort` | object | No | Sort configuration with `field` and `direction` properties |

**Sort Object:**

| Property | Type | Description |
|----------|------|-------------|
| `field` | string | Field to sort by (e.g., `created_at`, `score`, `status`) |
| `direction` | string | Sort direction: `asc` or `desc` |

**Response:**

```json
{
  "data": [
    {
      "run_id": "run_001",
      "batch_id": "abc123",
      "agent_id": "agent_456",
      "created_at": "2026-03-10T08:00:00Z",
      "status": "completed",
      "score": 0.92,
      "evaluator_results": [
        {
          "evaluator": "accuracy",
          "result": "pass"
        },
        {
          "evaluator": "toxicity",
          "result": "pass"
        }
      ],
      "inputs": {},
      "outputs": {}
    }
  ],
  "pagination": {
    "total": 42,
    "limit": 20,
    "offset": 0,
    "next_cursor": "eyJvZmZzZXQiOjIwfQ=="
  }
}
```

**Response Fields:**

| Field | Type | Description |
|-------|------|-------------|
| `data` | array | Array of evaluation run objects |
| `data[].run_id` | string | Unique identifier for the evaluation run |
| `data[].batch_id` | string | Batch identifier if run was part of a batch |
| `data[].agent_id` | string | Agent identifier |
| `data[].created_at` | string | ISO 8601 timestamp of when the run was created |
| `data[].status` | string | Run status: `completed`, `running`, or `failed` |
| `data[].score` | number | Overall score for the evaluation run (0-1) |
| `data[].evaluator_results` | array | Array of evaluator results with evaluator name and pass/fail result |
| `data[].inputs` | object | Input data used for the evaluation |
| `data[].outputs` | object | Output data from the evaluation |
| `pagination` | object | Pagination metadata |
| `pagination.total` | number | Total number of runs matching the filters |
| `pagination.limit` | number | Number of results per page |
| `pagination.offset` | number | Current offset in the result set |
| `pagination.next_cursor` | string | Cursor for fetching the next page (if available) |

## Code Examples

### Basic Request

Retrieve the most recent evaluation runs for an agent:

<Tabs>
  <Tab title="cURL">
    ```bash
    curl -X POST \
      -H "Content-Type: application/json" \
      -H "Authorization: YOUR_API_KEY" \
      -d '{"limit": 10, "sort": {"field": "created_at", "direction": "desc"}}' \
      https://api-f1db6c.stack.tryrelevance.com/latest/evals/agents/YOUR_AGENT_ID/runs
    ```
  </Tab>
  <Tab title="JavaScript">
    ```javascript
    const response = await fetch(
      'https://api-f1db6c.stack.tryrelevance.com/latest/evals/agents/YOUR_AGENT_ID/runs',
      {
        method: 'POST',
        headers: {
          'Content-Type': 'application/json',
          'Authorization': 'YOUR_API_KEY'
        },
        body: JSON.stringify({
          limit: 10,
          sort: {
            field: 'created_at',
            direction: 'desc'
          }
        })
      }
    );

    const data = await response.json();
    console.log(data);
    ```
  </Tab>
  <Tab title="Python">
    ```python
    import requests
    import json

    response = requests.post(
      'https://api-f1db6c.stack.tryrelevance.com/latest/evals/agents/YOUR_AGENT_ID/runs',
      headers={
        'Content-Type': 'application/json',
        'Authorization': 'YOUR_API_KEY'
      },
      data=json.dumps({
        'limit': 10,
        'sort': {
          'field': 'created_at',
          'direction': 'desc'
        }
      })
    )

    data = response.json()
    print(data)
    ```

## Pagination Best Practices

When working with large result sets, follow these best practices for efficient pagination:

<CardGroup cols={2}>
  <Card title="Use Cursors" icon="arrow-pointer">
    Prefer cursor-based pagination over offset-based pagination for better performance and consistency with large datasets.
  </Card>
  <Card title="Set Appropriate Limits" icon="gauge-high">
    Use reasonable page sizes (20-100 results) to balance performance and data transfer. Smaller pages are faster but require more requests.
  </Card>
  <Card title="Check Total Count" icon="calculator">
    Use the `pagination.total` field to determine the total number of results and calculate the number of pages needed.
  </Card>
  <Card title="Handle Empty Results" icon="circle-xmark">
    Always check if the `data` array is empty before processing results. An empty array indicates no more results are available.
  </Card>
</CardGroup>

### Pagination Example

Here's a complete example of paginating through all evaluation runs:

```javascript
async function getAllEvalRuns(agentId, apiKey) {
  const allRuns = [];
  let cursor = null;
  let hasMore = true;

  while (hasMore) {
    const body = {
      limit: 50,
      sort: { field: 'created_at', direction: 'desc' }
    };
    
    if (cursor) {
      body.cursor = cursor;
    }

    const response = await fetch(
      `https://api-f1db6c.stack.tryrelevance.com/latest/evals/agents/${agentId}/runs`,
      {
        method: 'POST',
        headers: {
          'Content-Type': 'application/json',
          'Authorization': apiKey
        },
        body: JSON.stringify(body)
      }
    );

    const data = await response.json();
    allRuns.push(...data.data);

    // Check if there are more results
    if (data.pagination.next_cursor) {
      cursor = data.pagination.next_cursor;
    } else {
      hasMore = false;
    }
  }

  return allRuns;
}
```

## Error Handling

The API returns standard HTTP status codes with descriptive error messages:

| Status Code | Error Type | Description |
|-------------|------------|-------------|
| **400** | Bad Request | Invalid parameters or missing required fields |
| **401** | Unauthorized | Missing or invalid authentication token |
| **403** | Forbidden | Insufficient permissions to access the agent |
| **404** | Not Found | Agent not found |
| **422** | Unprocessable Entity | Invalid request payload |
| **500** | Internal Server Error | Unexpected server error |

**Example Error Response:**

```json
{
  "error": {
    "code": 401,
    "message": "Invalid or missing authentication token"
  }
}
```

### Error Handling Example

```javascript
try {
  const response = await fetch(
    'https://api-f1db6c.stack.tryrelevance.com/latest/evals/agents/YOUR_AGENT_ID/runs',
    {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'Authorization': 'YOUR_API_KEY'
      },
      body: JSON.stringify({ limit: 20 })
    }
  );

  if (!response.ok) {
    const error = await response.json();
    throw new Error(`API Error ${error.error.code}: ${error.error.message}`);
  }

  const data = await response.json();
  // Process data
} catch (error) {
  console.error('Failed to fetch evaluation runs:', error.message);
}
```

## Frequently Asked Questions (FAQs)

<AccordionGroup>
  <Accordion title="What programming languages can I use with the Evals API?">
    You can use any programming language capable of making HTTP requests. The API is language-agnostic and works with cURL, JavaScript, Python, Ruby, Go, Java, and any other language with HTTP client libraries.
  </Accordion>
  
  <Accordion title="Are there rate limits for the Evals API?">
    Yes, API usage is subject to rate limits based on your subscription plan. If you exceed rate limits, you'll receive a 429 (Too Many Requests) response. Implement exponential backoff and respect rate limit headers in responses.
  </Accordion>
  
  <Accordion title="Can I filter by multiple criteria at once?">
    Yes, you can combine multiple filters in a single request. For example, you can filter by date range, status, and evaluator results simultaneously. All filters are applied with AND logic.
  </Accordion>
  
  <Accordion title="What's the difference between offset and cursor pagination?">
    Offset pagination skips a fixed number of results, while cursor pagination uses an opaque token to mark your position. Cursor pagination is more efficient and consistent for large datasets, especially when data is being added or modified.
  </Accordion>
  
  <Accordion title="How do I get my agent ID?">
    You can find your agent ID in the agent builder URL or in the agent settings. The agent ID is a unique identifier that looks like `agent_abc123xyz`.
  </Accordion>
  
  <Accordion title="Can I retrieve evaluation runs for multiple agents at once?">
    No, the current endpoint retrieves runs for a single agent at a time. To get runs for multiple agents, make separate API requests for each agent ID.
  </Accordion>
  
  <Accordion title="What timezone are the timestamps in?">
    All timestamps are in UTC (Coordinated Universal Time) and follow the ISO 8601 format (e.g., `2026-03-10T08:00:00Z`).
  </Accordion>
</AccordionGroup>
