# Agent Memory Snapshots

Memory snapshots are a powerful way to capture the entire state of your agents at any point in time. With this robust feature, you can effortlessly preserve conversation context, customise retention policies, and seamlessly restore previous states.

## Why Use Memory Snapshots

Memory snapshots help you leverage agent context across sessions. Whether you're debugging a long-running conversation or want to optimise your agents behaviour, snapshots supercharge your ability to manage state.

- Powerful state preservation across sessions
- Robust rollback capabilities
- Seamless integration with existing agents
- Flexible retention policies

## How To Create A Snapshot

To create a snapshot of your agents memory:

1. Navigate to your agent
2. Click the button in the top right
3. Select "Create Snapshot"
4. Enter a name and confirm

<Note>
Snapshots are retained for 30 days by default. After this period, they will be automatically deleted to optimise storage usage.
</Note>

## Restoring From A Snapshot

You can restore an agent to a previous state by selecting a snapshot from the snapshots panel. This will overwrite your agents current memory with the snapshot's contents.

<Note>
Restoring is irreversible. Your current agent state will be permanently replaced and there is no way to recover it.
</Note>

## Configuration Options

You can customise the following options for snapshot behaviour:

- Snapshot frequency
- Retention period
- Storage location
- Compression level

For more details on agent memory in general, see the [memory documentation](/build/agents/build-your-agent/memory).
