## What is Inventor

Inventor is a copilot agent that builds agents, surfaced inside the Relevance AI UI. It uses the Relevance AI MCP under the hood to take real actions on your project, so from a single chat surface you can build, debug, and iterate without leaving the platform.

## What you can do

<CardGroup cols={2}>
  <Card title="Build from a description" icon="wand-magic-sparkles">
    Create new agents, tools, workforces, and triggers from a natural-language description.
  </Card>
  <Card title="Debug failing agents" icon="bug">
    Inventor reads run history, identifies the failing step, explains what went wrong, and proposes a fix.
  </Card>
  <Card title="Iterate on existing assets" icon="pen-to-square">

  </Card>

## Inventor settings

Inventor settings tailor how Invent works for you and your project. They apply to Invent v3 (the Inventor panel and the `/invent/` page), not the legacy Blackbox tool Invent flow at `/tool/.../invent`.

### About You

About You is a free-text field of up to 5,000 characters, set per project. Set it once and it persists for that project. Inventor injects it into the model's user context to tailor tone, domain assumptions, and personalization.

Without About You, Inventor starts every conversation with zero context about who you are. With it, Inventor can:

<CardGroup cols={2}>
  <Card title="Tailor tone and language" icon="comments">
    Simpler explanations for non-technical users, more precise technical language for engineers.
  </Card>
  <Card title="Make smarter domain assumptions" icon="lightbulb">
    If you're in sales, it won't default to engineering-flavored tool designs.
  </Card>
  <Card title="Personalize responses naturally" icon="user">
    Addresses you by first name at natural moments like greetings and acknowledgements, not robotically in every message.
  </Card>
  <Card title="Reduce back-and-forth" icon="arrows-rotate">
    You don't have to re-explain your context every session.
  </Card>
</CardGroup>

It's descriptive only — it shapes how Inventor talks to you and what it assumes about your work, but it doesn't change Inventor's role or hardcode any tool configuration.

Describe who you are and what you're working on. For example:

- A recruiter sourcing candidates and screening applications.
- A developer building internal tooling and integrations.
- A marketer running campaigns and analyzing performance.

### Save and reference memories

Turn on this toggle to let Inventor remember specific facts across sessions — for example, "always return JSON, not markdown." This is separate from the static About You text: use About You for who you are, and memories for standing instructions you want Inventor to carry between sessions.

### Model

The model dropdown sets which LLM powers your Invent session. Choose from the configurable list; there's a default if you don't change it.
