## What are tools?

Tools are how you build integrations, LLM prompt chains or other step by step automations. You can build them in our no code tool builder. You can then share them as a form, run them in bulk on knowledge tables, or give them to your agents to help them complete work.

The possibilities are vast. You can tap into any API, choose from our extensive library of third-party integrations, or even incorporate custom code into your tools.

## Tool Sharing and Reusability

One of the key advantages of building tools in Relevance AI is their reusability across your project. Once you create a tool, you can:

### Benefits of Tool Sharing

The tool sharing UX makes it easy to:

- **Maintain consistency**: Use the same tool configuration across multiple agents and workflows
- **Save time**: Configure once, use everywhere—no need to rebuild similar tools
- **Collaborate effectively**: Team members can discover and use tools created by others
- **Update efficiently**: Changes to a tool propagate to all agents and workforces using it
- **Build modularly**: Create specialized tools that can be combined in different ways

<Tip>
When building tools, consider how they might be reused. Generic, well-documented tools are easier to share and integrate into different workflows.
</Tip>

### Use

Use provides you with an interface that allows you to run and test a tool as a shareable app.

### Build

The builder is where you create your tool. 

You can define the inputs expected of the tool, for example, a file input and a text input.

Then, select from a library of steps and add them to your tool.

Each step can be ran individually, which is useful for testing and debugging your tool.

### Logs

The Logs tab is where you can view the logs of your tool, which is useful for reviewing executions.

## How Tools Work with Agents

Tools extend your agents' capabilities by giving them the ability to perform actions. When you share a tool with an agent:

1. **The agent can access the tool**: The tool appears in the agent's available tools list
2. **Configuration is preserved**: Input settings, parameters, and logic remain consistent
3. **The agent decides when to use it**: Based on your instructions, the agent determines when the tool is needed
4. **Results are integrated**: Tool outputs are seamlessly incorporated into the agent's responses

This sharing mechanism allows you to build a library of reusable tools that can be mixed and matched across different agents and use cases.
