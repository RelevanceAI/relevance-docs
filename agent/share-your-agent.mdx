## Overview

Easily share your AI agent with others by choosing how they can interact with it. Whether you want to send a live chat link, embed a support widget, showcase tasks in an iframe, or allow full template cloning, you have flexible options to fit any use case.

You can also duplicate or clone agents for yourself to create variations, test changes, or move agents between projects.

---

## Duplicate or Clone Your Agent

Want to create a copy of your agent? You can duplicate agents using two methods:

### Copy to Project (Recommended)

The fastest way to duplicate an agent is using the **"Copy to Project"** feature. This method creates a complete copy of your agent in just a few clicks.

<Steps>
  <Step title="Go to your Agents list">
    Navigate to your Agents page in the Relevance AI platform
  </Step>
  <Step title="Find the agent you want to duplicate">
    Locate the agent you want to copy in your agents list
  </Step>
  <Step title="Open the agent menu">
    Click the three-dot menu (â‹®) next to the agent name
  </Step>
  <Step title="Select Copy to Project">
    Choose "Copy to Project" from the dropdown menu
  </Step>
  <Step title="Choose destination">
    Select the destination project (can be the same project or a different one)
  </Step>
  <Step title="Complete the copy">
    Click "Copy" to create your duplicate agent
  </Step>
</Steps>

#### Use Cases

- **Creating variations**: Build similar agents with slight modifications for different purposes
- **Testing changes**: Create a test version without affecting your production agent
- **Moving between projects**: Transfer agents between different projects or organizations (if you have access to multiple)
- **Backing up configurations**: Keep backup copies of important agents before making changes

### Using Clonable Templates

You can also duplicate your own agents using the clonable template feature described in the [Clonable Template](#clonable-template) section below. This method is useful when you want to share the agent with others AND keep copies for yourself, or when you need to share agents across different workspaces.

<Tip>
**Which method should you use?**
- Use **Copy to Project** for quick duplication within your workspace or between projects you have access to
- Use **Clonable Templates** when you want to share with external users or create a public template that can be cloned multiple times
</Tip>

## Clonable Template

Let others clone your Agent - complete with its instructions and setup - into their own workspace. This makes it easy to share best practices, pre-built use cases, or workflows without needing to rebuild from scratch. Cloned agents use the recipient's credits, not yours, making this perfect for scalable distribution.

<Tip>
**Duplicate your own agent:** You can use the clone link yourself to create a copy of your own agent. This is useful for creating variations, sharing across different workspaces, or when you want to create a template that can be cloned multiple times. For a faster method within the same workspace, use [Copy to Project](#copy-to-project-recommended) instead.
</Tip>
