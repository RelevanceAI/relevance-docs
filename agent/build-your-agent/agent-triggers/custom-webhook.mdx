## Webhook Triggers

Webhook triggers enable you to initiate agent tasks through HTTP requests from external systems. This powerful integration method allows you to connect virtually any third-party service to your Relevance AI agents without requiring native support.

### Key Features

- **Custom Webhook URLs**: Each webhook trigger provides a unique URL that receives incoming requests and activates your agent.
- **Flexible Payload Formatting**: You have complete control over how incoming data is processed. Use template variables with double brackets (e.g., `{{first_name}}`) to personalize messages or include the entire payload with `{{$}}`.
- **Thread ID Mapping**: For continuous conversations, map a specific field from your payload to serve as a Thread ID. This ensures new information is added to the same agent conversation, perfect for chatbot implementations.
- **Duplicate Prevention**: Map a Unique ID field to prevent processing the same event multiple times, ensuring data integrity and avoiding redundant operations.

### Setting Up a Webhook Trigger

1. Navigate to your agent's settings and select the Triggers section
2. Choose "Webhook Trigger" from the available options
3. Configure your webhook settings:
    - Customize the message template using variable placeholders
    - Map Thread ID and Unique ID fields as needed
    - Copy your unique webhook URL for integration with external systems

### Activating Your Webhook Trigger

<Warning>
Your webhook URL will return 404 errors until you complete both the save and publish steps below.
</Warning>

After configuring your webhook trigger, you must complete two critical steps before the endpoint becomes active:

<Steps>
  <Step title="Save the Trigger Configuration">
    Click the "Save" button to store your trigger configuration. This creates the webhook endpoint in the system.
  </Step>
  
  <Step title="Publish Your Agent">
    Click the "Publish" button in your agent's interface to make all configuration changes live. Publishing activates your webhook endpoint and makes it ready to receive requests.
  </Step>
  
  <Step title="Test Your Webhook URL">
    Once published, your webhook URL is active and ready to receive HTTP POST requests from external systems.
  </Step>
</Steps>

<Info>
Your webhook URL follows this format: `https://[region].stack.tryrelevance.com/agents/hooks/custom-trigger/[project_id]/[trigger_id]`
</Info>

The webhook endpoint only becomes active after both saving the trigger configuration and publishing your agent. Until then, any requests to the URL will return 404 errors.

### Troubleshooting Webhook 404 Errors

If you're receiving 404 errors when testing your webhook URL, follow this checklist:

<AccordionGroup>
  <Accordion title="Common Cause #1: Trigger Not Saved" icon="floppy-disk">
    **Problem**: The trigger configuration hasn't been saved to the database.
    
    **Solution**: 
    - Go to your agent's Triggers section
    - Verify your webhook trigger configuration is complete
    - Click the "Save" button to store the configuration
    - Wait a few seconds for the changes to propagate
  </Accordion>

  <Accordion title="Common Cause #2: Agent Not Published" icon="rocket">
    **Problem**: The agent hasn't been published, so the webhook endpoint isn't active.
    
    **Solution**:
    - Look for the "Publish" button in your agent's interface
    - Click "Publish" to make your agent and all triggers live
    - Wait for the publish process to complete (usually a few seconds)
    - Test your webhook URL again
  </Accordion>

  <Accordion title="Common Cause #3: Incorrect URL" icon="link">
    **Problem**: The webhook URL contains a typo or incorrect IDs.
    
    **Solution**:
    - Return to your agent's Triggers section
    - Copy the webhook URL directly from the trigger configuration
    - Verify the URL matches exactly (including region, project ID, and trigger ID)
    - Ensure there are no extra spaces or characters
  </Accordion>
</AccordionGroup>

**Quick Resolution Checklist:**

1. ✅ Trigger configuration is complete
2. ✅ Trigger has been saved
3. ✅ Agent has been published
4. ✅ Webhook URL is copied correctly
5. ✅ External system is sending POST requests (not GET)

### Implementation Examples

Webhook triggers can be used for various automation scenarios:

- Trigger customer support responses when new tickets are created
- Process form submissions from your website
- Respond to events from third-party platforms
- Create automated workflows between different systems

## FAQs

<AccordionGroup>
  <Accordion title="Can I use webhook triggers with any external system?">
    Yes, any system capable of sending HTTP POST requests can integrate with Relevance AI's webhook triggers, making them extremely versatile for various integration scenarios.
  </Accordion>

  <Accordion title="How secure are webhook triggers?">
    Webhook triggers use unique URLs that are difficult to guess. For additional security, you can implement Unique ID mapping to prevent duplicate processing and validate incoming requests.
  </Accordion>

  <Accordion title="What happens if the same webhook is triggered multiple times?">
    By mapping a Unique ID field, you can prevent duplicate processing. If an event with the same Unique ID is received again, it will be ignored, ensuring data integrity.
  </Accordion>

  <Accordion title="Why am I getting 404 errors on my webhook URL?">
    404 errors typically occur when either the trigger hasn't been saved or the agent hasn't been published. Both steps are required for the webhook endpoint to become active. See the "Troubleshooting Webhook 404 Errors" section above for detailed resolution steps.
  </Accordion>

  <Accordion title="Do I need to republish my agent every time I change the webhook configuration?">
    Yes, any changes to your agent's configuration (including trigger settings) require republishing for the changes to take effect.
  </Accordion>
</AccordionGroup>
