## Overview

Interaction Webhooks enable your agents to respond to commands in real-time during live meetings. When participants mention trigger words in the meeting chat, your agent can immediately take action‚Äîpausing recording, performing searches, sending messages, or executing any tool in its toolkit.

This powerful feature transforms your Meeting Notetaker from a passive recorder into an active meeting participant that responds to your needs in real-time.

## What Are Realtime "On Call" Commands?

Realtime "On Call" Commands allow meeting participants to interact with your agent during live meetings by typing trigger words in the meeting chat. The agent listens for these keywords and responds immediately with configured actions.

## How It Works

<Steps>
  <Step title="Trigger Word Detection">
    Your agent monitors the meeting chat for configured trigger words (e.g., "@notetaker", "hey bot")
  </Step>
  
  <Step title="Command Parsing">
    When a trigger word is detected, the agent parses the command to understand the requested action
  </Step>
  
  <Step title="Action Execution">
    The agent executes the appropriate action using its available tools (search, pause/resume, send message, etc.)
  </Step>
  
  <Step title="Response Delivery">
    The agent responds in the meeting chat or takes the requested action immediately
  </Step>
</Steps>

## Setting Up Interaction Webhooks

### Prerequisites

- A Meeting Notetaker Agent with RecallAI integration
- Access to Advanced Settings in your agent configuration

### Configuration Steps

<Steps>
  <Step title="Navigate to Advanced Settings">
    1. Open your Meeting Notetaker Agent
    2. Go to **Advanced Settings** in the agent configuration
    3. Find the **Interaction Webhooks** section
  </Step>
  
  <Step title="Enable Webhooks">
    1. Toggle **Enable Interaction Webhooks** to ON
    2. This activates real-time command listening during meetings
  </Step>
  
  <Step title="Configure Trigger Words">
    Set up the keywords that will activate your agent:
    
    ```json
    {
      "trigger_words": [
        "@notetaker",
        "hey bot",
        "assistant"
      ]
    }
    ```
    
    <Tip>
    Use distinctive trigger words that won't be accidentally mentioned in normal conversation.
    </Tip>
  </Step>
  
  <Step title="Define Actions">
    Configure what actions are available and how they're triggered:
    
    ```json
    {
      "actions": {
        "pause": {
          "keywords": ["pause", "stop recording"],
          "tool": "pause_recording",
          "confirmation": "Recording paused"
        },
        "resume": {
          "keywords": ["resume", "start recording"],
          "tool": "resume_recording",
          "confirmation": "Recording resumed"
        },
        "search": {
          "keywords": ["search", "look up", "find"],
          "tool": "google_search",
          "response_format": "chat_message"
        },
        "note": {
          "keywords": ["note", "remember", "action item"],
          "tool": "add_priority_note",
          "response_format": "confirmation"
        }
      }
    }
    ```
  </Step>
  
  <Step title="Test Your Setup">
    1. Create a test meeting
    2. Have your agent join
    3. Try trigger words in the chat
    4. Verify the agent responds correctly
  </Step>
</Steps>

## Available Actions

### Recording Control

Control recording during sensitive discussions:

<Tabs>
  <Tab title="Pause Recording">
    **Command**: `@notetaker pause recording`
    
    **What it does**:
    - Immediately pauses audio/video recording
    - Stops transcription
    - Maintains meeting presence
    - Sends confirmation message
    
    **Use case**: Discussing confidential information that shouldn't be recorded
  </Tab>
  
  <Tab title="Resume Recording">
    **Command**: `@notetaker resume recording`
    
    **What it does**:
    - Resumes audio/video recording
    - Restarts transcription
    - Sends confirmation message
    
    **Use case**: Continue recording after sensitive discussion ends
  </Tab>
</Tabs>

### Live Search & Research

Perform searches during meetings:

<Tabs>
  <Tab title="Google Search">
    **Command**: `@notetaker search [query]`
    
    **Example**: `@notetaker search latest AI regulations`
    
    **What it does**:
    - Performs Google search with the query
    - Extracts top results
    - Posts summary in meeting chat
    - Includes relevant links
    
    **Use case**: Quick fact-checking or research during discussions
  </Tab>
  
  <Tab title="Knowledge Base Search">
    **Command**: `@notetaker find [topic]`
    
    **Example**: `@notetaker find pricing policy`
    
    **What it does**:
    - Searches your agent's knowledge base
    - Retrieves relevant documents
    - Shares key information in chat
    
    **Use case**: Access company documentation during meetings
  </Tab>
</Tabs>

### Communication Actions

Send messages and share resources:

<Tabs>
  <Tab title="Send Message">
    **Command**: `@notetaker send [message]`
    
    **Example**: `@notetaker send link to product roadmap`
    
    **What it does**:
    - Retrieves requested resource
    - Posts in meeting chat
    - Makes it accessible to all participants
    
    **Use case**: Share documents or links without leaving the meeting
  </Tab>
  
  <Tab title="Notify Team">
    **Command**: `@notetaker notify [team/person]`
    
    **Example**: `@notetaker notify engineering team`
    
    **What it does**:
    - Sends notification to specified team/person
    - Includes meeting context
    - Can trigger escalation workflows
    
    **Use case**: Loop in stakeholders who aren't in the meeting
  </Tab>
</Tabs>

### Note-Taking Actions

Capture important information on-the-fly:

<Tabs>
  <Tab title="Priority Note">
    **Command**: `@notetaker note: [content]`
    
    **Example**: `@notetaker note: Follow up with legal team by Friday`
    
    **What it does**:
    - Adds to priority action items list
    - Tags with timestamp and speaker
    - Can trigger immediate notifications
    
    **Use case**: Capture critical action items as they're discussed
  </Tab>
  
  <Tab title="Decision Log">
    **Command**: `@notetaker decision: [content]`
    
    **Example**: `@notetaker decision: Moving forward with Option B`
    
    **What it does**:
    - Logs as key decision
    - Captures context and rationale
    - Highlights in meeting summary
    
    **Use case**: Document important decisions with full context
  </Tab>
</Tabs>

### Tool Integration

Connect to any tool in your agent's toolkit:

<AccordionGroup>
  <Accordion title="Notion Integration" icon="n">
    **Command**: `@notetaker add to notion [content]`
    
    **What it does**:
    - Creates Notion page or database entry
    - Includes meeting context
    - Links back to meeting recording
    
    **Use case**: Document decisions in your Notion workspace in real-time
  </Accordion>
  
  <Accordion title="Slack Integration" icon="slack">
    **Command**: `@notetaker post to slack [channel]`
    
    **What it does**:
    - Posts message to specified Slack channel
    - Includes meeting context and participants
    - Can @mention relevant people
    
    **Use case**: Share meeting updates with teams not in attendance
  </Accordion>
  
  <Accordion title="CRM Integration" icon="address-book">
    **Command**: `@notetaker log to crm`
    
    **What it does**:
    - Creates activity in CRM (Salesforce, HubSpot)
    - Logs meeting notes and action items
    - Updates deal or contact records
    
    **Use case**: Keep CRM updated during customer calls
  </Accordion>
  
  <Accordion title="Project Management" icon="list-check">
    **Command**: `@notetaker create task [description]`
    
    **What it does**:
    - Creates task in project management tool (Jira, Asana, Trello)
    - Assigns based on meeting context
    - Sets due dates if mentioned
    
    **Use case**: Convert action items to tasks immediately
  </Accordion>
</AccordionGroup>

## Advanced Configuration

### Custom Command Patterns

Create sophisticated command patterns with variables:

```json
{
  "commands": [
    {
      "pattern": "@notetaker search {query}",
      "action": "google_search",
      "parameters": {
        "query": "{query}",
        "num_results": 5
      }
    },
    {
      "pattern": "@notetaker remind {person} about {topic}",
      "action": "send_reminder",
      "parameters": {
        "recipient": "{person}",
        "subject": "{topic}",
        "context": "meeting_transcript"
      }
    },
    {
      "pattern": "@notetaker schedule followup in {timeframe}",
      "action": "create_calendar_event",
      "parameters": {
        "title": "Follow-up Meeting",
        "duration": "30 minutes",
        "relative_time": "{timeframe}"
      }
    }
  ]
}
```

### Conditional Actions

Execute actions based on meeting context:

```json
{
  "conditional_actions": {
    "customer_meeting": {
      "auto_log_to_crm": true,
      "notify_account_team": true,
      "priority_notes": "high"
    },
    "internal_meeting": {
      "auto_log_to_crm": false,
      "notify_account_team": false,
      "priority_notes": "normal"
    }
  }
}
```

### Response Templates

Customize how your agent responds:

```json
{
  "response_templates": {
    "pause_confirmation": "‚úÖ Recording paused. Type '@notetaker resume' to continue.",
    "search_intro": "üîç Searching for: {query}...",
    "search_results": "Here's what I found:\n{results}",
    "action_item_added": "‚úì Added to action items: {content}",
    "error": "‚ùå Sorry, I couldn't complete that action. Please try again."
  }
}
```

## Best Practices

<AccordionGroup>
  <Accordion title="Choose Clear Trigger Words" icon="bullseye">
    Select trigger words that are:
    - **Distinctive**: Won't be mentioned accidentally
    - **Easy to remember**: Simple for participants to use
    - **Professional**: Appropriate for business meetings
    
    ‚úÖ Good: "@notetaker", "@assistant", "hey bot"
    ‚ùå Avoid: "um", "okay", "yes" (too common)
  </Accordion>
  
  <Accordion title="Provide Command Help" icon="circle-question">
    Make commands discoverable:
    - Have agent announce available commands when joining
    - Respond to "@notetaker help" with command list
    - Include command reference in meeting invitations
    
    **Example help message**:
    ```
    Available commands:
    ‚Ä¢ @notetaker pause/resume - Control recording
    ‚Ä¢ @notetaker search [query] - Google search
    ‚Ä¢ @notetaker note: [text] - Add action item
    ‚Ä¢ @notetaker help - Show this message
    ```
  </Accordion>
  
  <Accordion title="Confirm Actions" icon="check">
    Always provide confirmation when actions are executed:
    - Visual confirmation in chat
    - Clear success/error messages
    - Include next steps when relevant
    
    This ensures participants know their commands were received and processed.
  </Accordion>
  
  <Accordion title="Handle Errors Gracefully" icon="triangle-exclamation">
    When commands fail:
    - Provide clear error messages
    - Suggest alternatives
    - Offer to escalate to human if needed
    
    **Example**: "I couldn't find that document. Did you mean 'product roadmap Q1'?"
  </Accordion>
  
  <Accordion title="Respect Privacy" icon="user-shield">
    For recording control commands:
    - Respond immediately to pause requests
    - Confirm when recording resumes
    - Never record without explicit consent
    - Log all pause/resume actions for compliance
  </Accordion>
</AccordionGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Agent Not Responding to Commands" icon="circle-xmark">
    **Possible Causes:**
    - Interaction Webhooks not enabled
    - Trigger word not configured correctly
    - Chat monitoring not working on platform
    
    **Solutions:**
    - Verify Interaction Webhooks are enabled in Advanced Settings
    - Check trigger word spelling and configuration
    - Test with different trigger words
    - Ensure agent has chat access permissions
  </Accordion>
  
  <Accordion title="Commands Executing Incorrectly" icon="triangle-exclamation">
    **Possible Causes:**
    - Command pattern not matching input
    - Tool configuration issues
    - Insufficient permissions
    
    **Solutions:**
    - Review command pattern syntax
    - Test commands in isolation
    - Verify tool permissions and configuration
    - Check agent logs for error details
  </Accordion>
  
  <Accordion title="Delayed Responses" icon="clock">
    **Possible Causes:**
    - Network latency
    - Complex tool execution
    - High system load
    
    **Solutions:**
    - Optimize tool performance
    - Use simpler commands for time-sensitive actions
    - Consider async processing for complex tasks
    - Provide "processing" acknowledgment immediately
  </Accordion>
</AccordionGroup>

## Security Considerations

<Warning>
**Important**: Interaction Webhooks execute actions in real-time during meetings. Ensure proper access controls and validation.
</Warning>

### Access Control

- **Authenticate Commands**: Verify commands come from authorized participants
- **Limit Actions**: Restrict sensitive actions to specific users
- **Audit Logging**: Log all commands and actions for compliance
- **Rate Limiting**: Prevent command spam or abuse

### Data Privacy

- **Sensitive Information**: Be cautious with commands that expose data
- **Recording Control**: Always honor pause/resume requests immediately
- **Consent**: Ensure all participants consent to interactive agent features
- **Compliance**: Follow data protection regulations (GDPR, CCPA)
